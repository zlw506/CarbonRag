# V1.4.8 开发日志

负责席位：#3

## 本轮目标

建设 CarbonRag 碳因子数据库基础版，以 CarbonStop CCDB 中国碳数据库公开页面为强制主参考，形成可检索、可维护、可被后续碳核算调用的运行时因子库。

## 实际完成内容

- 新增 OpenSpec change：`carbon-factor-database-foundation`，明确 CarbonStop CCDB 公开页面/公开网关为本轮主参考，并写清非公开/商业加密值不得硬编入库。
- 新增运行时表：`carbon_factor_sources`、`carbon_factor_records`、`carbon_factor_aliases`、`carbon_factor_import_jobs`，覆盖 SQLite 与 PostgreSQL schema。
- 新增 CarbonStop CCDB public adapter，导入公开分类、公开行快照和可计算公开因子；对“加密数据，点击咨询购买”的展示值只记录跳过摘要，不进入计算库。
- 新增后端碳因子 API：用户侧搜索、详情、来源、facets；管理员侧导入、CarbonStop 同步、启停维护接口。
- 新增运行时 FactorRegistry 读取逻辑，运行时数据库因子优先，内置文件种子作为兜底。
- 新增前端“碳因子库”页面，支持一级分类、二级分类、年份筛选、关键词搜索、卡片列表和详情抽屉。
- 修复 CarbonStop 分类映射问题：一级分类写入 `industry`，二级分类写入 `category`；分类数量按实际可计算公开因子统计，避免“官网二级分类有数量但查询结果为 0”。

## 已知限制

- 本轮只把 CarbonStop CCDB 公开页面中可计算的公开数值纳入运行时因子库；公开页面显示为加密/咨询购买的值不会进入计算库。
- 前端已提供管理员同步入口所需 API，但尚未做完整管理员维护 UI。
- 碳核算页面尚未把每条因子引用深链到“碳因子库”详情页，后续可继续打磨。

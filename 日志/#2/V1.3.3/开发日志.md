# V1.3.3 #2 开发日志

## 本轮目标

把政策知识三段式摄取从“后端可验证”推进到“产品可演示”：管理员能在现有产品界面中运行内置合成展示源，查看任务、工作流、治理 metadata、分块和检索命中。展示源必须明确标记为 demo/showcase，不伪装成官方政策证据。

## 实际完成内容

- 新增 OpenSpec change：`policy-ingestion-showcase-readiness`。
- 新增内置离线展示源 `low-carbon-campus-action`，包含演示 HTML、内部 source URI、标题、演示编号、发布日期、施行日期和低碳/碳核算/节能改造等政策术语。
- 根据 #1 PR review 修正：展示源不再使用 fake `gov.cn`、fake 国务院/中国政府网、fake 官方文号或官方版权，并以 `visibility=demo`、`public_policy_demo` 标识进入检索与引用链路。
- 新增管理员 API：
  - `GET /api/v1/admin/policy-sources`
  - `POST /api/v1/admin/policy-sources/{source_id}/run`
  - `GET /api/v1/admin/policy-sources/{source_id}/status`
  - `GET /api/v1/admin/policy-sources/{source_id}/chunks`
  - `GET /api/v1/admin/policy-sources/{source_id}/retrieval-preview`
- 管理员控制台 `/admin` 集成“政策知识三段式摄取”区域，不新增单独 demo 页。
- 运行入口使用真实链路：`crawl_ingest -> policy_ingest -> parse -> normalize metadata -> chunks -> public_policy_demo retrieval`。
- 重复运行同一内置政策源时刷新同一个 `public_policy_web` 条目，避免无限重复创建。
- 检索预览返回 `public_policy_demo` evidence shape；真实官方 `public_policy_web` 仍按 `public_policy` 处理，不改变 `/ask`、RAG Lab、retrieval-only、calc/report/session 默认行为。
- 新增产品路径验证脚本：`scripts/verify_policy_showcase.py`。
- 忽略本地运行生成的 policy staging 目录，避免提交运行产物。

## 已知限制

- 当前展示入口使用内置离线合成样例，不启用公网 live crawler、不启用 Scrapyd 调度，也不能作为官方政策依据引用。
- 当前只提供一个 curated source，后续可扩展为官方白名单 source catalog。
- 检索预览为 BM25/runtime chunk 验证，不代表完整生产级向量检索或图谱检索。
- 本轮不接 Docling、MinerU、OFDRW、Scrapyd 生产调度，也不改变默认问答/碳核算主流程。

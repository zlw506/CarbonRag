## #1 约束

本轮按 #1 在 Mattermost 的 constrained ACK 执行：

- 不默认自动运行公网政策爬虫。
- 默认只初始化官方白名单源目录和 Admin 控制面。
- 管理员可手动触发抓取；自动定时仅作为显式配置预留。
- Scrapy 缺失或关闭时应用启动不失败，状态显示为 unavailable/disabled。
- 抓取结果只能进入 `pending_review` 候选区。
- 管理员发布前不得进入 `public_policy_web`，不得进入 `/ask` 默认检索，不得伪装官方 citation。

## 已完成

- 新增 runtime 表：
  - `policy_crawl_sources`
  - `policy_crawl_runs`
  - `policy_crawl_candidates`
- SQLite/PostgreSQL bootstrap 均已覆盖，使用 additive `CREATE TABLE IF NOT EXISTS`。
- 新增 `PolicyCrawlerScheduler`：
  - startup 幂等初始化官方源；
  - shutdown 可停止调度线程；
  - 默认 scheduled mode 关闭；
  - 非重入保护；
  - Scrapy unavailable/disabled 安全降级；
  - robots、depth、pages、delay、concurrency、timeout、user-agent、failure backoff 进入 safe limits。
- 新增 Admin API：
  - `GET /api/v1/admin/policy-crawler/status`
  - `GET /api/v1/admin/policy-crawler/sources`
  - `POST /api/v1/admin/policy-crawler/sources/{source_id}/run`
  - `GET /api/v1/admin/policy-crawler/runs`
  - `GET /api/v1/admin/policy-crawler/candidates`
  - `POST /api/v1/admin/policy-crawler/candidates/{candidate_id}/publish`
  - `POST /api/v1/admin/policy-crawler/candidates/{candidate_id}/reject`
- Admin 页面新增“实时政策爬虫”区域，展示 provider、调度模式、官方源、运行记录、候选文档和发布/拒绝入口。
- 新增 smoke 脚本：`scripts/verify_policy_live_crawler.py`。

## 已知限制

- 默认不运行自动公网调度；需要显式配置才会进入 scheduled mode。
- 本轮不接 Scrapyd、不做分布式调度、不做浏览器自动化。
- 本轮不抓任意 URL，只保留官方白名单源。
- 发布候选只入队 `crawl_ingest`，后续索引仍由现有知识任务 runner 处理。

# V1.3.4 手动抓取入口可用性修复记录

## 背景

- 本地演示发现 Scrapy provider 仍被旧的 `RAG_ENABLE_POLICY_CRAWLER=false` 默认开关挡住。
- 结果是 Admin 手动模式开启时，状态仍可能显示 provider 未启用，用户无法顺利验证官方源手动抓取。

## 修复

- `PolicyCrawlerScheduler` 默认创建 `ScrapyCrawlerProvider` 时改用 `rag_policy_live_crawler_manual_enabled`。
- “管理员手动触发”和“自动定时公网调度”已解耦。
- 自动定时公网调度仍保持默认关闭，继续遵守 #1 constrained ACK。
- Admin 页面把“手动抓取官方源”按钮放入每个官方源条目正文区域。
- Scrapy 不可用时 Admin 页面显示明确提示，点击后只记录 `unavailable` 运行结果，不影响 `/ask`。

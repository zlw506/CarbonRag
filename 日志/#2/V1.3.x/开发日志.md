# V1.3.x #2 开发日志

## 本轮版本号与负责席位

- 版本号：V1.3.x
- 负责席位：#2 / tbx
- 分支：`t2/v1.3/rag-cache-refresh-smoke-fix`

## 本轮目标

修复 V1.4.6 融合后本地 smoke 发现的 RAG 运行态缓存刷新问题：用户上传私有文档并完成知识入库索引后，RAG Lab / retrieval-only API 应能立即检索到该用户显式选择的个人知识条目，同时继续保持跨用户隔离。

## 实际完成内容

- 在知识入库完成后的 private/mixed retriever 缓存清理路径中，同步清理 `get_rag_engine_service` 单例缓存。
- 在文件上传、用户知识刷新、admin 知识刷新和后台知识刷新路径中补齐 RAG engine cache clear。
- 新增回归测试，确保知识缓存刷新不会只清 retriever，而漏掉 RAG engine 单例。
- 保持 `/ask`、RAG Lab 响应结构、BM25 fallback、用户数据隔离和碳核算流程不变。

- OpenSpec change-id：`ragpdf-parser-chunker-adapter`

## 本轮目标

借鉴本地 `ragPdfSystem.zip` 中成熟的多格式文档解析、表格文本化、页码/幻灯片结构保留和评测 smoke 思路，增强 CarbonRag 现有 parser/chunker 能力，并用本地闭环脚本验证“文档 fixture -> 知识任务 -> 分块索引 -> RAG 检索 evidence”可跑通。

## 实际完成内容

- 增强 `app.knowledge.parsers.parse_document`：
  - 支持 HTML、CSV、XLSX、XLS、DOCX、PDF、PPTX 的结构化文本提取。
  - PDF 输出 `[Page N]` 页码标记。
  - CSV/Excel/DOCX 表格输出表格/工作表/行号标记。
  - PPTX 在可选 `python-pptx` 可用时输出 `[Slide N]` 幻灯片标记。
  - 旧版 `.doc` / `.ppt` 返回明确错误，不静默失败。
- 增强 `DefaultParserProvider`：
  - 从页码、表格、工作表、幻灯片标记生成 `DocumentBlock`。
  - metadata 增加 `block_count`、`page_count`、`table_count`、`text_length`。
  - 不改变现有 `/ask`、RAG Lab、retrieval-only、calc/report/session 默认行为。
- 新增闭环验证脚本：
  - `scripts/verify_ragpdf_closed_loop.py`
  - 使用临时 SQLite、临时用户、临时上传文件记录，不写入正式 runtime 数据。
  - 验证 HTML/CSV/Markdown fixture 能进入知识任务、生成 chunks，并通过 RAG engine private retrieval 返回 references。
- 新增测试：
  - `backend/tests/test_ragpdf_closed_loop.py`
  - 扩展 `backend/tests/test_rag_parser_provider.py`

## 已知限制

- 本轮只迁移 ragPdfSystem 中适合 CarbonRag 当前架构的 parser/chunker 与本地验证思路，不迁移 Milvus、Celery、MinIO、Vue 页面、私有虚拟环境或样例数据。
- PPTX 解析依赖 `python-pptx`，属于 optional dependency；未安装时返回明确错误。
- 复杂扫描 PDF、OCR、Docling/MinerU/OFDRW 仍走已有可选 provider/adapter 路线，本轮不把它们变成强依赖。
- V1.5.x main 已新增 #1 的 session-file-reading workbench；本轮没有改动该前端/会话入口，只通过回归测试确认没有破坏。

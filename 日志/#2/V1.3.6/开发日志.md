# V1.3.6 #2 开发日志

## 本轮目标

接入 `scrapy/scrapy` 与 `scrapy/scrapyd` 的政策爬虫能力边界，把 CarbonRag 政策采集层从单一 local Scrapy provider 扩展为 local Scrapy / optional Scrapyd backend。

## 上游确认

- `scrapy/scrapy`：Python Web crawling / scraping framework，BSD-3-Clause。
- `scrapy/scrapyd`：运行 Scrapy spiders 的 service daemon，BSD-3-Clause。
- 用户提供的 `scrapy/scrap` 不是有效 GitHub 仓库，本轮不依赖该地址。

## 施工边界

- 不整仓复制 Scrapy / Scrapyd 源码，只通过依赖和 Adapter / Provider 接入。
- 不默认自动运行公网爬虫，仍只允许管理员手动触发。
- 不抓任意 URL，只允许官方白名单源，并保留 robots、限速、深度、页数和超时限制。
- 抓取结果只进入 `pending_review`，发布前不进入 `public_policy_web` 和 `/ask` 默认检索。

# V1.4.5 开发日志

## 基本信息

- 负责席位：#1 首席开发团队
- 版本：V1.4.5
- 日期：2026-05-07
- OpenSpec change-id：`carbon-inventory-trust-baseline`
- 主线：V1.4.x 碳核算

## 本轮目标

把 V1.4.4 的 factor-driven 演示计算器升级为企业基础碳核算可信底座，使每个计算数字都能追溯到：

- 原始活动数据
- 因子来源
- 单位换算
- 公式 trace
- 因子快照
- 证据字段
- scope summary

本轮不改前端 UI，不扩复杂行业公式，不做完整 Scope 3，不做国家因子库自动同步。

## 实际完成

- 新增 `carbon_inventories`、`carbon_activity_items`、`carbon_calculation_lines`、`carbon_factor_snapshots`、`carbon_evidence_references`、`carbon_inventory_summaries`。
- 保留 `carbon_calculations` 兼容旧 report/session 查询，并补充 `inventory_id`、原始 activity JSON、scope summary 和 factor counts。
- `POST /api/v1/calc-carbon` 继续兼容旧三字段请求；旧字段只转换非零 activity，避免只填电力时出现天然气/柴油无关 warning。
- `CarbonActivityItem` 新增 `province`、`data_quality`、`evidence_reference`、`source_document_id`、`entry_method`、`requested_factor_id`。
- 新增 `data/factors/electricity_cn_2023_official.json`，结构化录入全国、剩余组合、化石能源、七大区域、31 个省级 2023 电力因子。
- 新增 `data/factors/fuel_combustion_cn_guidance_seed.json`，把燃料缺省值标为 `guidance_default`，不伪装为 official。
- `FactorRegistry` 支持用户指定因子、官方地区/年份命中、上级地区 fallback、guidance fallback、demo fallback。
- 响应新增 `inventory_id`、`total_kgco2e`、`scope_summary`、`activity_count`、`official_factor_count`、`fallback_factor_count`。
- 新增 OpenSpec change：`openspec/changes/carbon-inventory-trust-baseline/`。
- 更新上游数据源决策文档，明确 CEADs、MEE/NBS、电力因子、国家因子库、guidance seed 的边界。

## 已知限制

- Scope 2 market-based / 绿电凭证口径仍是预留，不把绿电任意算零。
- 国家温室气体排放因子数据库本轮只做 manual import 方向设计，不做自动同步。
- 燃料 `guidance_default` 只适合基础演示或缺少更权威因子时参考，企业正式核算应优先使用国家因子库或适用行业指南。
- 前端碳核算页暂未展示 V1.4.5 新增的全部 inventory 字段，这属于后续 UI 轮次。

# V1.5.6 开发日志

负责席位：#1

## 本轮目标

继续修 AskPage 聊天体验：降低用户消息气泡饱和度，并修复上传附件后历史文件全部出现在输入框底栏的问题。

## 实际完成

- 将主题 token 中的 `chatUserBubble` / `chatUserBubbleAlt` 从实心主题色改为低透明度主题色。
- 用户消息文字颜色改回主题正文色，避免浅色低饱和气泡上出现过亮反差。
- 移除“已解析文件自动加入本轮草稿”的逻辑。
- 静默轮询会话文件状态时，只保留当前草稿里已经明确选中的上传文件，不再把历史 ready 文件塞进 composer。
- 上传新文件后，仅把本次上传返回的 `file_id` 加入本轮附件草稿。

## 验证结果

- `openspec validate --all`：通过，31 passed。
- `npm run typecheck`：通过。
- `npm run build`：通过，保留既有 Vite chunk size warning。
- GitNexus：当前索引 stale，仅作为结构定位参考；AskPage upstream impact 为 LOW。

## 已知限制

- 本轮只修 composer 草稿区自动混入历史附件的问题，不新增“历史文件选择器”。
- 会话后台文件仍由后端保留；是否默认检索全部会话文件由后续文件读取策略继续收敛。

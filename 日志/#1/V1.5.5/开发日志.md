# V1.5.5 开发日志

负责席位：#1

## 本轮目标

修复 AskPage 两个聊天体验问题：流式回答阶段 Markdown 原字符堆叠，以及上传附件文件名过长撑开 chip。

## 实际完成

- 助手消息渲染前增加轻量 Markdown 归一化，补齐缺失的标题、列表、编号列表空格和换行。
- Markdown 归一化跳过 fenced code block，避免误改代码块内容。
- 流式中和完成后使用同一套 Markdown 归一化规则，减少完成瞬间的排版突变。
- composer 附件 chip 与用户消息附件 chip 改为固定宽度。
- 附件文件名区域增加 `min-width: 0` 与 `text-overflow: ellipsis`，超长文件名以省略号截断。

## 验证结果

- `npm run typecheck`：通过。
- `npm run build`：通过，保留既有 Vite chunk size warning。
- `git diff --check`：无格式错误，仅有 Windows CRLF 提示。
- GitNexus：当前索引为 stale，仅作为结构提示；AskPage 影响链路为前端路由/入口，未触及后端、RAG、碳核算或部署。

## 已知限制

- 本轮是前端渲染层修正，不改变模型实际输出文本。
- 对极端不规范 Markdown 的修复仍是轻量级，不替代后端输出格式约束。

## 后续交接

- 后续聊天 UI 改动必须继续保证“流式中”和“完成后”排版心智一致。
- 后续附件 chip 不允许再恢复完整长文件名撑开输入区的样式。

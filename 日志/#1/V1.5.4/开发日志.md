# V1.5.4 开发日志

负责席位：#1

## 本轮目标

重做 V1.5.3 不合格的 AskPage 附件交互，让上传文件更接近 ChatGPT 式聊天体验。

## 实际完成

- 移除“会话文件”中间卡片。
- 上传后文件直接进入输入框下方的草稿附件区。
- 草稿附件支持 `X` 移除，移除后不会因轮询再次出现。
- 发送时只提交已解析完成的草稿附件。
- 发送后附件跟随用户消息展示在消息流中。
- 历史消息通过助手回答里的 `private_upload` citation，反推显示上一条用户消息所使用的文件。

## 验证结果

- `npm run typecheck`：通过。
- `npm run build`：通过，保留既有 chunk size warning。
- `git diff --check`：无格式错误，仅有 Windows CRLF 提示。
- `gitnexus detect_changes`：HIGH，原因是 AskPage 核心聊天页影响面较大；本轮实际未改后端、RAG、碳因子或部署。

## 已知限制

- 历史消息附件展示依赖 assistant citation 反推；后续如需强持久化，应在消息表增加 message attachments 元数据。
- 点 `X` 是从当前草稿移除，不等同于删除服务器文件。

## 后续交接

- #2/#3 不要恢复 V1.5.3 的“会话文件卡片”交互。
- 后续文件读取体验继续以“composer draft attachments -> user message attachments -> assistant citations”为基线。

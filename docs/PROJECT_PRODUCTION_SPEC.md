## 3. 项目是什么

CarbonRag 是一个面向中小企业的“双碳”政策与企业应用智能问答系统。

它不是普通聊天机器人，也不是泛化知识助手，而是一个面向垂直业务场景的 RAG 系统。它要把以下能力整合起来：

- 公共政策与标准知识检索
- 企业私有资料接入
- 基础碳核算
- 报告草稿生成
- 后续可扩展的安全与合规能力

本项目当前的产品目标，不是“回答一切问题”，而是围绕“双碳政策理解—企业数据利用—核算输出—报告生成”形成第一条可演示业务闭环。

---

## 4. 项目要解决的核心问题

### 4.1 业务问题本质

当前双碳相关政策、标准、补贴指南更新频繁，文本分散，口径复杂。中小企业通常缺少专门团队持续跟踪、解读、整理并映射到自身经营数据，导致以下问题长期存在：

- 政策看不懂
- 更新跟不上
- 企业自己的数据用不起来
- 碳核算成本高、解释难
- 报告和申报材料整理成本高
- 通用大模型回答不够可靠、不够可追溯

### 4.2 本项目的工程目标

因此，CarbonRag 当前阶段要解决的不是“做一个问答框”，而是解决下面这个工程问题：

> 如何在时间有限的前提下，先构建一个能够基于权威政策文本和企业样例数据，完成检索问答、基础碳核算和报告生成的最小闭环系统，并保证该系统后续可继续向本地化、私有化和更高合规方向演进。

## 5. 当前项目服务对象

本项目当前主要面向以下对象：

### 5.1 直接服务对象
- 缺少双碳专业团队的中小企业
- 需要快速理解政策、标准、补贴和申报要求的企业管理者或业务人员
- 需要利用企业内部能耗、生产、物流等数据进行基础碳核算的使用者

- 后续企业内部管理与审计协同人员

## 9. 当前技术栈基线（冻结版）

本节是当前阶段的工程统一口径。  
若后续要改，必须由项目总监批准后再改文档，再改实现。

### 9.1 技术栈目标原则
- 优先稳定
- 优先可演示
- 优先低复杂度
- 优先后续可替换
- 不为“未来理想形态”牺牲当前施工效率

### 9.2 前端基线
- 前端形态：Web 前端
- 语言：TypeScript
- 框架：React
- 构建工具：Vite
- UI 方案：Ant Design
- 状态管理：优先轻量方案，不先引入复杂状态框架
- 网络请求：Axios
- 页面目标：问答页、碳核算页、报告页、管理占位页

### 9.3 后端基线
- 语言：Python
- 框架：FastAPI
- 原因：适合 API、RAG、数据处理、碳核算与后续模型接入统一

### 9.4 检索层基线
- 文档切分：后端服务内实现
- Embedding：先通过可替换封装层接入
- 向量检索：首轮优先轻量可控方案
- 原则：先打通，再谈大规模向量库替换

### 9.5 数据层基线
- 公共知识数据：`data/public/`
- 企业脱敏样例：`data/private_sample/`
- 排放因子与计算模板：`data/factors/`
- 结果产物：`data/outputs/`

### 9.6 模型层基线
- 当前统一：第三方云端 API
- 强制要求：统一封装，不准写死供应商
- 配置外置：模型名、API Base URL、Key、温度、上下文等全部外置

### 9.7 环境变量纪律
- 仓库中只允许提交 `.env.example`
- 不允许提交真实 API Key
- 不允许把 key 写进源码
- 不允许把 key 写进 README 示例正文
- 不允许把 key 发到共享提交记录里

## 12. Git 纪律（强制执行）

### 12.1 总原则

Git 不是备份工具，而是基线控制工具。

所有 Git 操作必须服务于四个目标：

1. 知道当前基线是什么
2. 知道这一轮改了什么
3. 改坏后能快速回退
4. 演示结果能对应到具体提交

### 12.2 强制规则

- 不允许在脏工作树上继续叠改
- 不允许不做 checkpoint 就开始一轮正式施工
- 不允许高风险改动直接落 `main`
- 不允许把多个主题混进一个提交
- 不允许把真实敏感数据提交进仓库
- 不允许把 API Key、企业隐私数据写入版本历史
- 不允许 hotfix 顺手改架构

### 12.3 分支规则

- `main`：只放稳定、可演示、可交付内容
- `dev`：当前主开发集成分支
- `feature/*`：单功能施工分支
- `hotfix/*`：最小修复分支
- `verify/*`：历史验证分支

### 12.4 提交流程

开工前必须：

1. 确认工作树干净
2. 同步当前基线
3. 留 checkpoint
4. 从 `dev` 切出功能分支
5. 在该分支完成本轮任务

### 12.5 提交信息模板

- checkpoint：`checkpoint: before v0.x.x`
- 正常提交：`v0.x.x: <本轮内容摘要>`
- 热修提交：`v0.x.x: minimal <bugfix> on stable baseline`
- 恢复提交：`restore: revert <module/path> to <baseline>`

# CarbonRag V1.1.8B UX Fix Mapping

## 1. 聊天区不突出，仍像企业后台工作台
- 优先级：P0
- 当前状态：仍存在，V1.1.7 已部分缓解
- 对应页面/模块：Ask 工作台壳层、会话栏、依据抽屉、上下文胶囊
- 最可能责任文件：
  - `frontend/src/pages/AskPage/index.tsx`
  - `frontend/src/layouts/AppShell.tsx`
  - `frontend/src/styles/global.css`
- 修复方式：
  - 继续收窄左侧会话栏
  - Ask 页默认专注模式
  - 右侧依据/状态默认不常驻
  - 中间消息列最大化
- 验收标准：
  - 首屏一眼能看出聊天区是主视觉区
  - 用户无需先理解侧栏和指标也能开始对话

## 2. 输入框体验差且易出错
- 优先级：P0
- 当前状态：仍存在
- 对应页面/模块：composer、快捷键、发送行为
- 最可能责任文件：
  - `frontend/src/pages/AskPage/index.tsx`
  - `frontend/src/styles/global.css`
- 修复方式：
  - composer 固定在底部可见区域
  - 主 row 只保留附件、输入框、发送
  - `Enter` 发送、`Shift+Enter` 换行
  - `Ctrl+A` 只作用于文本域，不再误选整页
- 验收标准：
  - 输入框始终可见
  - 键盘快捷键行为符合主流聊天工具预期
  - 不再出现因误选整页导致的“问题不能为空”

## 3. 思考态和流式反馈存在但体感弱
- 优先级：P0
- 当前状态：仍存在，V1.1.4 / V1.1.5 已完成基础状态机
- 对应页面/模块：assistant 消息态、streaming 事件消费、SSE 事件顺序
- 最可能责任文件：
  - `frontend/src/pages/AskPage/index.tsx`
  - `frontend/src/styles/global.css`
  - `backend/app/api/v1/endpoints/sessions.py`
- 修复方式：
  - 占位消息更早出现
  - thinking 与 streaming 动画和标签明确区分
  - 无 reasoning chunk 时也要有稳定 thinking 状态
  - metadata 收尾保持稳定
- 验收标准：
  - 用户能明显感知“思考中 -> 正在输出 -> 已完成”
  - 回复不再像整段延迟块刷新

## 4. 工程信息噪音太多
- 优先级：P0
- 当前状态：仍存在
- 对应页面/模块：context capsule、系统连通状态、trace/details
- 最可能责任文件：
  - `frontend/src/pages/AskPage/index.tsx`
  - `frontend/src/components/SystemInfoPanel.tsx`
  - `frontend/src/styles/global.css`
- 修复方式：
  - Ask 顶部只保留单个上下文胶囊
  - 系统状态移入抽屉中的折叠区
  - 普通页面移除常驻 `SystemInfoPanel`
  - trace / feedback 下沉到 details
- 验收标准：
  - 默认视图优先阅读回答
  - 工程信息只在按需查看时出现

## 5. 报告页不可用
- 优先级：P0
- 当前状态：仍存在
- 对应页面/模块：会话默认选择、来源默认回退、报告生成提示
- 最可能责任文件：
  - `frontend/src/pages/ReportPage/index.tsx`
  - `backend/app/report/service.py`
  - `backend/tests/test_report_service.py`
  - `backend/tests/test_report_route.py`
- 修复方式：
  - 前端默认选当前 session 和最近合法来源
  - 无来源时给清晰引导，不再形成禁用死路
  - 后端在 `source_message_ids` 为空时回退到当前 session 最近合法来源
  - `carbon_summary` 可回退到最近一条 carbon result
- 验收标准：
  - 当前 session 下可直接生成报告
  - 缺来源时页面能明确告诉用户下一步做什么

## 6. 管理员入口过于暴露
- 优先级：P0
- 当前状态：仍存在，权限本身已受控
- 对应页面/模块：全局导航、用户菜单、管理员角色呈现
- 最可能责任文件：
  - `frontend/src/layouts/AppShell.tsx`
  - `frontend/src/constants/navigation.ts`
  - `frontend/src/router/index.tsx`
- 修复方式：
  - 普通主导航移除管理员入口
  - 管理员通过头像菜单或次级入口进入后台
  - Ask 首屏保留角色态，但弱化后台感
- 验收标准：
  - 普通用户看不到管理员入口
  - 管理员能力不再污染普通聊天体验

## 7. 会话列表信息碎片化
- 优先级：P0
- 当前状态：仍存在，V1.1.5 已做收窄
- 对应页面/模块：session rail、session card
- 最可能责任文件：
  - `frontend/src/pages/AskPage/index.tsx`
  - `frontend/src/styles/global.css`
- 修复方式：
  - 标题作为一级信息
  - 更新时间和消息数作为二级信息
  - 附件数、知识条目数弱化到次级行/tooltip
- 验收标准：
  - 会话列表可快速扫读
  - 选中态一眼可见

## 8. 碳核算页缺少用户导向
- 优先级：P1
- 当前状态：仍存在
- 对应页面/模块：表单说明、结果区层级、因子依据展示
- 最可能责任文件：
  - `frontend/src/pages/CarbonCalcPage/index.tsx`
  - `frontend/src/styles/global.css`
- 修复方式：
  - 字段名和帮助文本更用户化
  - 结果区先显示结论，再展开明细和因子依据
  - 系统状态不再常驻
- 验收标准：
  - 普通用户能理解输入项和结果区
  - 先看结论，再按需展开细节

## 9. 知识库页面过于工程化
- 优先级：P1
- 当前状态：仍存在
- 对应页面/模块：我的知识库工作区、任务摘要、上传/知识条目展示
- 最可能责任文件：
  - `frontend/src/pages/MyKnowledgePage/index.tsx`
  - `frontend/src/styles/global.css`
- 修复方式：
  - 从表格优先改成内容卡片优先
  - 先展示“内容 / 状态 / 可操作项”
  - 任务指标退后，不再占普通用户主视线
- 验收标准：
  - 页面更像用户资料页，而不是后台任务页
  - 用户能一眼知道自己有什么知识内容

## 10. 视觉风格与响应式不统一
- 优先级：P1
- 当前状态：仍存在
- 对应页面/模块：Ask / Report / Calc / Knowledge / AppShell / 全局样式
- 最可能责任文件：
  - `frontend/src/styles/global.css`
  - `frontend/src/layouts/AppShell.tsx`
  - 各页面 `index.tsx`
- 修复方式：
  - 统一字号、间距、Tag/Chip 语义、按钮密度
  - 统一抽屉、空态、Alert 语气
  - 1152px 和更窄宽度下优先保证聊天区
- 验收标准：
  - 窄屏不再三栏拥挤
  - 不同页面的视觉语言明显更统一

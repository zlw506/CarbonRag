# V1.6.5 RAG-Pro 本地模型与 Parity Hardening

## Summary

本轮 `change-id: rag-pro-local-model-and-parity-hardening`，目标是把 V1.6.4 的 RAG-Pro 骨架继续推进到本地可验证闭环：BGE-M3 dense/sparse embedding、bge-reranker、Milvus Lite/Milvus、知识库文档索引、Test QA、Ask trace。

## Default Direction

默认路线回到 RAG-Pro 本地模型：

```env
RAG_EMBEDDING_PROVIDER=bge_m3
RAG_MODEL_AUTO_DOWNLOAD=false
RAG_HF_ENDPOINT=https://hf-mirror.com
RAG_MODEL_CACHE_DIR=./data/outputs/models
RAG_VECTOR_BACKEND=milvus_lite
RAG_MILVUS_URI=./data/outputs/milvus_lite/carbonrag.db
RAG_RERANK_PROVIDER=bge_reranker
RAG_RERANK_MODEL=BAAI/bge-reranker-v2-m3
RAG_EMBEDDING_DEVICE=cpu
```

`openai_compatible` embedding 只作为高级选项保留：只有团队确实拥有 `/v1/embeddings` endpoint 与 embedding model 时才启用。普通聊天 API 不能替代 embedding API。

## Validation Criteria

本轮最低验收不是“接口存在”，而是：

- BGE-M3 输出 `dense=1024` 维。
- BGE-M3 sparse lexical weights 非空。
- bge-reranker-v2-m3 模型存在并可加载，或明确报缺失。
- Milvus Lite/Milvus 能创建 collection/db。
- KnowledgeBase -> Document -> Chunk -> index -> search 真实命中。
- `/api/v1/rag/test-qa` 和 AskPage 选择知识库时返回 trace 与 citation。
- 如果 BGE/Milvus/reranker 不可用，必须返回 degraded/failed，不允许显示 RAG 成功。

## Windows Limitation

截至本轮，Windows pip 源无法安装可用的 `milvus-lite` wheel。Windows 本机可以完成 BGE-M3/reranker 下载和 embedding smoke，但 Milvus Lite 文件型 smoke 可能仍需要 WSL/Linux、Linux CI、或远程 Milvus 服务。

这不是理由去回退 fake vector。Windows 如果无法跑 Milvus Lite，必须把 blocker 写清楚，然后在支持平台完成真实验收。

V1.6.8 起，Windows 默认验收路线改为 Docker Milvus Standalone：`RAG_VECTOR_BACKEND=milvus`、`RAG_MILVUS_URI=http://127.0.0.1:19530`。Milvus Lite `.db` 路线只保留给 WSL/Linux/macOS。

官方核验口径：

- Milvus 文档说明：在 `pymilvus` 中把 `MilvusClient` 的 `uri` 指向本地 `.db` 文件时，会使用 Milvus Lite，并在当前目录生成本地数据库文件。参考：https://milvus.io/docs/milvus_lite.md
- PyPI `milvus-lite` 页面说明当前支持 Ubuntu 与 macOS，并明确 `Windows is not yet supported`；下载文件也只有 manylinux 与 macOS wheel。参考：https://pypi.org/project/milvus-lite/
- 本机 Windows + Python 3.11.15 执行 `pip index versions milvus-lite` 与 `pip install "milvus-lite>=2.5.1"` 都返回 `No matching distribution found`。
- 因此本机失败点不是“没写安装命令”，而是当前 Windows 环境拿不到匹配的 `milvus-lite` 发行包。
- RAG-Pro 能使用 Milvus Lite 的关键不是特殊魔法，而是通过 `pymilvus.MilvusClient(uri=settings.MILVUS_URI)` 使用本地文件 URI；它的默认 `MILVUS_URI=backend/data/milvus.db`，在支持平台上即可触发 Milvus Lite。

后续团队建议：

- Windows 开发者只在本机跑 BGE-M3 / reranker / Memory fallback smoke。
- 需要 Milvus Lite 真实验收时，优先用 WSL2、Linux 机器、Linux CI、或远程 Milvus 服务。
- 不允许因为 Windows 本机装不上 `milvus-lite` 就恢复 hash embedding 或假向量成功。

## C Drive Rule

所有模型和 Hugging Face 缓存必须落到 `data/outputs` 或其他大容量盘：

```env
RAG_MODEL_CACHE_DIR=F:\Project\CarbonRag\data\outputs\models
HF_HOME=F:\Project\CarbonRag\data\outputs\hf-cache
HUGGINGFACE_HUB_CACHE=F:\Project\CarbonRag\data\outputs\hf-cache\hub
```

禁止模型落到：

- `C:\Users\<user>\.cache\huggingface`
- `backend/app/models`
- `backend/backend/app/models`
- 任何可能被 Git 追踪的源码目录

推荐打包命令示例：

```powershell
New-Item -ItemType Directory -Force F:\CarbonRag-share | Out-Null
Compress-Archive -Path data\outputs\models\BAAI\bge-m3 -DestinationPath F:\CarbonRag-share\bge-m3.zip -Force
Compress-Archive -Path data\outputs\models\BAAI\bge-reranker-v2-m3 -DestinationPath F:\CarbonRag-share\bge-reranker-v2-m3.zip -Force
```

如果本机安装了 7-Zip，更推荐用 7z 分卷压缩，避免超大 zip 传输中断：

```powershell
7z a -v1024m F:\CarbonRag-share\bge-m3.7z data\outputs\models\BAAI\bge-m3
7z a -v1024m F:\CarbonRag-share\bge-reranker-v2-m3.7z data\outputs\models\BAAI\bge-reranker-v2-m3
```

团队收到模型后放回：

```text
F:\Project\CarbonRag\data\outputs\models\BAAI\bge-m3
F:\Project\CarbonRag\data\outputs\models\BAAI\bge-reranker-v2-m3
```

从云端 `main` 拉取代码的团队必须按这个目录结构放置离线包，不能放到 `backend/app/models`、`backend/models`、`C:\Users\<user>\.cache\huggingface` 或仓库源码目录：

```text
CarbonRag/
└─ data/
   └─ outputs/
      └─ models/
         └─ BAAI/
            ├─ bge-m3/
            │  ├─ config.json
            │  ├─ pytorch_model.bin
            │  ├─ tokenizer.json
            │  └─ ...
            └─ bge-reranker-v2-m3/
               ├─ config.json
               ├─ model.safetensors
               ├─ tokenizer.json
               └─ ...
```

如果收到的是 zip/7z 包，解压后必须保证最终目录名就是 `bge-m3` 和 `bge-reranker-v2-m3`，不要多套一层 `bge-m3\bge-m3\`。

然后运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/rag-smoke-local-model.ps1
```

验证时至少应看到 BGE-M3 smoke 输出类似：

```text
bge_m3 2 1024 True
```

其中 `1024` 表示 dense vector 维度正确，`True` 表示 sparse lexical weights 非空。

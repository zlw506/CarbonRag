# V1.6.8 Milvus Docker Runtime 与 RAG-Pro 闭环验收

## Summary

本轮 `change-id: rag-pro-milvus-runtime-profile-hardening`，目标是把 Windows 交付路线从 native `milvus_lite + .db` 切到 Docker Milvus Standalone，消除 Windows 无法安装 `milvus-lite` 的阻断。

CarbonRag 继续以 RAG-Pro 为主复现对象。RAG 开发阶段准许完整迁移/改写 RAG-Pro 的核心结构与逻辑，但不得提交 `3rdparty` 源码树，并必须保留 CarbonRag 现有 auth/session/AskPage/report/OpenSpec/GitNexus/Mattermost 边界。

## Runtime Profiles

### Windows Docker

默认比赛验收路线：

```env
RAG_VECTOR_BACKEND=milvus
RAG_MILVUS_URI=http://127.0.0.1:19530
RAG_REQUIRE_REAL_VECTOR=true
RAG_EMBEDDING_PROVIDER=bge_m3
RAG_MODEL_AUTO_DOWNLOAD=false
RAG_MODEL_CACHE_DIR=./data/outputs/models
RAG_RERANK_PROVIDER=bge_reranker
RAG_RERANK_MODEL=BAAI/bge-reranker-v2-m3
```

启动：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/rag-start-milvus-docker-windows.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/rag-smoke-milvus-standalone.ps1
```

停止：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File scripts/rag-stop-milvus-docker-windows.ps1
```

### WSL/Linux/macOS Milvus Lite

只在支持 `milvus-lite` 的平台使用：

```env
RAG_VECTOR_BACKEND=milvus_lite
RAG_MILVUS_URI=./data/outputs/milvus_lite/carbonrag.db
```

RAG-Pro 的 `.db` 文件 URI 方式本质上依赖 `pymilvus.MilvusClient(uri="...db")` 触发 Milvus Lite。该路线不是 native Windows 方案。

### Memory Dev

`memory` 仅用于开发调试：

```env
RAG_VECTOR_BACKEND=memory
RAG_REQUIRE_REAL_VECTOR=false
```

该 profile 必须显示 `vector_runtime=memory_dev` 且 `degraded=true`，不能作为比赛验收。

## Model Package Location

云端 `main` 不提交模型。#1 离线发送的模型包必须解压到：

```text
data/outputs/models/BAAI/bge-m3
data/outputs/models/BAAI/bge-reranker-v2-m3
```

不要放到：

- `C:\Users\<user>\.cache\huggingface`
- `backend/app/models`
- `backend/models`
- 任何源码目录

## Docker Disk Space Warning

Milvus Docker 镜像、容器和 volume 由 Docker Desktop 管理，默认可能占用系统盘或 WSL 虚拟磁盘。C 盘空间不足时，必须先在 Docker Desktop / WSL 层把数据目录迁移到空间充足的磁盘，再拉取 Milvus 镜像。

## Acceptance

最低验收必须跑完整链路：

1. Docker Milvus Standalone 可连接 `http://127.0.0.1:19530`。
2. BGE-M3 输出 1024 维 dense vector 且 sparse weights 非空。
3. 创建 KnowledgeBase。
4. 上传或创建真实文档。
5. parse -> chunk -> index 写入 Milvus Standalone。
6. `/api/v1/rag/search` 命中文档并返回 `vector_runtime=milvus_standalone`。
7. `/api/v1/rag/test-qa` 返回 trace 与 citation。
8. AskPage 选择 KB 提问并显示 citation。

只走 memory/BM25 fallback、trace 显示错误 runtime、BGE 模型缺失、Docker 19530 不可连，任一项都不能宣称 V1.6.8 完成。

## Notes

- 2C2G VPS 不跑 Milvus；云端 Milvus 后续需要 4C/8GB 以上或独立向量服务。
- 普通聊天 API 不能替代 `/v1/embeddings` API。默认继续使用本地 BGE-M3。
- #1 Windows 主机已将 Docker Desktop 数据目录迁到 `F:\AcademicHub\Docker\vm-data` 与 `F:\AcademicHub\Docker\wsl\DockerDesktopWSL`。其他团队必须先把 Docker 数据盘放到空间充足的磁盘，再拉取 Milvus 镜像。
- `scripts/rag-pro-real-vector-smoke.ps1` 已作为 V1.6.8 硬验收：必须出现 `dense_count>=1`、`sparse_count>=1`、`rerank_applied=true`、`vector_runtime=milvus_standalone`、`degraded=false`。
- 官方参考：Milvus Windows Docker 文档、Milvus Standalone requirements、milvus-lite PyPI。

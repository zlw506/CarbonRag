# V1.7.3 官方政策正文抽取与 RAG 入库可靠性修复

## Summary
- 本轮 `change-id: crawler-real-content-extraction-and-rag-ingestion-fix`，只修官方政策网页正文抽取、artifact 预览、质量评分和 RAG 入库门禁。
- 固定验收 URL：`https://www.gov.cn/zhengce/content/202604/content_7066483.htm`，必须抽取到标题、文号、发布日期和不少于 2000 个中文字符正文。
- 发布、提交、聊天室/公告消息必须带版本号 `V1.7.3`；不触碰当前未跟踪的 `docs/碳核算专项/`、`演示/`、`docs/PLAN/V1.7.2B.md`。

## Key Changes
- 先新增审查报告 `docs/research/crawler/GOV_CN_POLICY_EXTRACTION_AUDIT_V1.7.3.md`，记录现有超时、Scrapy no documents、urllib raw HTML fallback、raw/cleaned/markdown 文件大小、预览为空、质量分误判、duplicate 展示和 RAG 入库前后 chunk 数据。
- 修前端超时：`dryRunPolicyCrawlerSource`、`runPolicyCrawlerSource`、`publishPolicyCrawlerCandidateToRag` 使用 crawler 专用 120s timeout；错误提示展示 crawler 阶段，不再被全局 30s httpClient 硬杀。
- 新增 gov.cn 正文抽取器：`backend/app/knowledge/extractors/gov_cn_policy.py`，使用 Scrapy 依赖里的 `parsel.Selector` 解析 `#UCAP-CONTENT`、`.TRS_Editor`、`.pages_content`、`.article`、`.conTxt`、`.content` 等区域，并保留锚点 fallback。
- 改 artifact 写入：每个候选目录固定写 `raw.html`、`cleaned.txt`、`document.md`、`extraction.json`；cleaned/markdown 小于 800 字符时标记 `extract_empty_text` 或 `extract_empty_markdown`，质量分不得高于 40，禁止发布到 RAG。
- 新增 admin artifacts 接口：`GET /api/v1/admin/policy-crawler/candidates/{candidate_id}/artifacts`，返回 raw/cleaned/markdown 是否存在、大小、预览、estimated_chunk_count、artifact_errors。
- FilePreviewDrawer 对 `crawler_candidate` 使用 artifacts 数据展示 Markdown、纯文本、raw HTML 摘要和已发布后的 RAG chunks，避免 metadata 有路径但抽屉显示空。
- 质量分拆分为 `extraction_quality_score`、`topic_relevance_score`、`topic_class`；保留 `candidate_quality_score` 兼容旧 UI，但不再把间接低碳相关政策显示成“核心双碳质量 100”。
- duplicate 候选改 UI 与规则：`duplicate_content_hash` 显示“重复内容”；已有 `rag_doc_id` 时提供“查看已有 RAG 文档”，无 `rag_doc_id` 时按钮文案为“重新入库”，但空 artifact 仍禁止入库。
- `publish-to-rag` 增加强制门禁：markdown/cleaned 均存在且 >= 800 字符、estimated_chunk_count > 0、extraction_quality_score >= 60；quick pipeline 后必须 `chunk_count > 0`、`indexed_chunk_count > 0`、`rag_search_smoke_passed=true`，否则不得标记 published。

## Public APIs / Interfaces
- 新增 `GET /api/v1/admin/policy-crawler/candidates/{candidate_id}/artifacts`，仅 admin 可访问。
- Candidate summary 增加 `extraction_quality_score`、`topic_relevance_score`、`topic_class`、`artifact_errors`、`cleaned_size`、`markdown_size`、`estimated_chunk_count`。
- `POST /api/v1/admin/policy-crawler/candidates/{candidate_id}/publish-to-rag` 低质量或空抽取时返回 400，detail 明确 `empty_extraction`、`zero_chunks`、`index_failed` 或 `search_smoke_failed`。
- 现有 `GET /api/v1/file-previews/crawler_candidate/{candidate_id}` 保持兼容，但 crawler 预览优先读取 artifact 目录。

## Assumptions
- 本轮不做新 UI 美化、不扩推荐源库、不做定时任务、不引入完整 Crawl4AI/Crawlab。
- CI 使用 gov.cn fixture；live smoke 访问真实 URL，网络不可用时记录 blocker，但不能影响 fixture 回归。
- `parsel` 来自 Scrapy 依赖，不新增额外 HTML 解析依赖。
- 合格标准是：后台预览看到 `国务院关于推进服务业扩能提质的意见`、`国发〔2026〕7号`、`2026年04月21日` 和有效正文；发布到 RAG 后搜索 `服务业 扩能提质 节能环保服务` 命中该候选文档。

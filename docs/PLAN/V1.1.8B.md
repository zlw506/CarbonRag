# CarbonRag V1.1.8B UX Audit Issue-to-Code Mapping + Remediation

## Summary
- 本轮只做 UX 审查整改，不扩新业务。基线按当前 `main` 的 V1.1.7 / V1.1.6F 能力推进，重点修复《CarbonRag UX 审查》里仍成立的 P0 / P1 问题。
- 这轮的真实责任分布已经比较清楚：
  - `AskPage` 仍是一号责任区，当前聊天布局、消息态、composer、context capsule、依据抽屉基本都集中在这里。
  - `SystemInfoPanel` 是工程信息噪音的主要来源，因为它被直接挂在 Ask / Report / Calc / MyKnowledge 多页。
  - `ReportPage` 已有自动选择当前会话和默认来源逻辑，所以“报告页不可用”更像是**来源资格判断过严 + 默认引导不足 + 死路状态处理差**，不是缺模块。
  - 管理员路由本身已经做了前后端权限拦截，当前问题主要是**入口和后台感暴露在普通聊天主路径里**，不是权限完全失守。
- 已锁定的产品决策：
  - Ask 页默认 `focus mode`
  - 右侧依据 / 系统状态继续保留，但默认更隐身，按需打开
  - 管理员入口与系统信息本轮采用你确认的策略：**页面内折叠**
  - 本轮只允许最多 `1` 个只读子 agent；若实现期启用，只用于审查对照或测试探查，不做多写者并发
  - 所有新增或修改的用户可见文案保持中文

## Key Changes
- 先落“问题到代码责任区”的映射文档：
  - 新增 `docs/ux/V1.1.8B_UX_FIX_MAPPING.md`
  - 固定按审查问题逐条写：
    - 审查问题
    - 优先级
    - 对应页面/模块
    - 最可能责任文件
    - 修复方式
    - 修复后验收标准
  - 这份文档要把当前状态标清：`仍存在 / 部分缓解 / 已由 V1.1.7 部分覆盖`，避免重复修已经做完的点

- Ask / 聊天工作台整改，第一责任区：
  - 责任文件优先锁定：
    - `frontend/src/pages/AskPage/index.tsx`
    - `frontend/src/layouts/AppShell.tsx`
    - `frontend/src/components/SystemInfoPanel.tsx`
    - `frontend/src/styles/*`
    - `frontend/src/services/sessions.ts`
    - `backend/app/api/v1/endpoints/sessions.py`
    - `backend/app/ai_runtime/runtime/*`
  - 具体整改：
    - 中间聊天列继续放大，成为绝对主区；左侧 session rail 再收窄并支持更明确的 collapsed / mini 模式
    - 右侧依据 / 状态继续用 `Drawer`，默认关闭；只保留轻量入口和依据数量，不再常驻大面积信息层
    - 底部 composer 重做成真正的 sticky/floating 输入栏；主 row 只保留附件、输入框、发送，scope / 上传 / 挂接状态压成紧凑 chips 或次级行
    - 修复快捷键和输入区误操作：
      - `Enter` 发送
      - `Shift+Enter` 换行
      - `Ctrl+A` 只在输入框聚焦时作用于文本域，不再误选整页
      - 空输入校验只在真实提交时触发，不因焦点误切换触发“问题不能为空”
    - assistant 消息态继续保留 `pending / thinking / streaming / done / error`，但视觉反馈要明显增强：
      - 占位更早出现
      - thinking 与 streaming 的标签、动画、节奏明确区分
      - thinking chunk 若存在则放在更紧凑的可折叠区域；不存在也要有明显 pulse / dots / heartbeat
    - citations 默认折叠；消息卡片正文优先，依据详情只在抽屉内集中查看
    - trace、token、summary、context 等工程信息不再占首屏主区：
      - Ask 顶部只保留单个上下文胶囊
      - 展开后才显示详细上下文组成、压缩状态、token 估算
      - `SystemInfoPanel` 从 Ask 主流程退到抽屉中的“系统状态”分区

- Report 页修复，目标是“当前 session 可直接生成报告”：
  - 责任文件优先锁定：
    - `frontend/src/pages/ReportPage/index.tsx`
    - `frontend/src/services/reports.ts`
    - `backend/app/report/service.py`
  - 修复方向：
    - 打通“当前 session 直接进入可用状态”的默认链路：
      - 默认选中当前 session
      - 默认选中最近一条符合模板要求的 assistant 消息
      - `carbon_summary` 默认选中最近一条 carbon result
    - 去掉禁用死路；如果当前 session 缺少合法来源，不是只禁用，而是给明确引导：
      - 当前缺什么来源
      - 应先去问答页还是去碳核算页
      - 一键跳转入口
    - 后端 `ReportService` 增加更稳的默认来源容错：
      - 当 `source_message_ids` 为空时，可按 report type 尝试回退到“当前 session 最近合法来源”
      - 如果仍不满足条件，再返回明确 422，而不是让前端落入不可理解状态
    - 页面说明从工程话术改成用户导向，不再让“来源选择规则”成为理解门槛

- Calc 页、知识库页、管理员入口做产品化收口，不动业务主线：
  - `frontend/src/pages/CarbonCalcPage/index.tsx`
    - 字段名和说明进一步用户化，单位解释和示例前置
    - 结果区改成“先结论、后明细、因子折叠”
    - 右侧系统信息不常驻，转为折叠/抽屉
  - `frontend/src/pages/MyKnowledgePage/index.tsx`
    - 从表格优先改成用户卡片优先
    - 优先展示“内容 / 状态 / 可操作项”
    - ingest / rebuild / retry 等任务指标降级，只给管理员或二级展开看
    - `SystemInfoPanel` 不再作为普通用户知识页的常驻右栏
  - `frontend/src/layouts/AppShell.tsx` + `frontend/src/constants/navigation.ts` + `frontend/src/router/index.tsx`
    - 普通用户继续看不到管理员入口
    - 管理员入口从高显著主路径退后：
      - Ask focus mode 下不在主聊天路径高亮出现
      - 只通过页面内折叠入口、头像菜单或次级入口进入
    - 管理员身份标识保留，但弱化，不让聊天首屏继续像后台系统

- 会话栏、视觉系统、响应式统一：
  - 会话卡片信息分三级：
    - 一级：标题
    - 二级：最后更新时间 / 简短摘要
    - 三级：消息数 / 附件数 / 知识数，弱化到 hover 或次级行
  - 统一跨页样式：
    - 字号层级
    - 卡片密度
    - 按钮样式
    - Tag/Chip 颜色语义
    - 抽屉 / Popover / Empty / Alert 的统一语气
  - 响应式口径锁定：
    - 1440px：session rail 默认展开但更窄，evidence drawer 默认关闭
    - 1152px：session rail 默认更轻或自动 collapsed，主聊天区优先
    - 更窄宽度：全局导航和 session rail 都退到可展开模式，避免“三栏压迫感”

- 后端只做同步补缺，不重写业务：
  - `ask/stream` 契约不扩面，继续沿用现有 SSE 事件
  - 需要确认并补齐：
    - `status=thinking` 一定早于正文 delta
    - 无 reasoning chunk 时仍给稳定 thinking 状态
    - `metadata` 一定带 `citations`、`source_summary`、`memory_state`
    - flush / chunk 节奏不过度缓慢，不让前端只看到“分段大块刷新”
  - `auth / role` 只做回归确认：
    - `/auth/me` 的角色字段被前端正确消费
    - 管理员入口不只是 UI 隐藏，路由和后端权限仍严格

## Public APIs / Interfaces
- 不新增新的业务 API。
- 继续沿用：
  - `POST /api/v1/sessions/{id}/ask`
  - `POST /api/v1/sessions/{id}/ask/stream`
  - 现有 report / calc / knowledge / admin 路由
- 本轮允许的行为级变化：
  - Ask 前端默认优先走 `/ask/stream`，`/ask` 只作失败降级
  - `POST /api/v1/reports` 在“当前 session 有唯一或明显默认来源”时可自动回退来源，不再让前端因空来源直接落入死路
- 新增文档接口：
  - `docs/ux/V1.1.8B_UX_FIX_MAPPING.md`

## Test Plan
- Ask / 聊天页 UX 回归：
  - 首屏聊天区成为视觉中心
  - 左侧 session rail 更窄且可折叠
  - 右侧依据 / 系统状态默认不常驻
  - 输入框始终固定在底部可见区域
  - `Ctrl+A / Enter / Shift+Enter` 行为符合聊天工具预期
  - 用户消息靠右、AI 靠左、system 弱化
  - assistant 明显经历 `pending -> thinking -> streaming -> done/error`
  - 上下文胶囊默认轻量，展开后才显示详细 context 组成

- 报告页回归：
  - 进入当前 session 后可直接生成可用报告
  - 无合法来源时不出现禁用死路，而是有清晰引导
  - `policy_summary / mixed_analysis / carbon_summary` 各自的默认来源选择和错误提示正确

- Calc / Knowledge / Admin 回归：
  - Calc 页结论优先、明细后置、因子可折叠
  - Knowledge 页默认更像用户页面而不是任务后台
  - 普通用户看不到管理员入口；管理员入口在普通聊天主路径中明显退后
  - `SystemInfoPanel` 不再在普通用户页面大面积常驻

- 后端 / 契约回归：
  - `/api/v1/sessions/{id}/ask/stream` 在有 reasoning、无 reasoning、provider error 三种情况下事件顺序稳定
  - `/api/v1/reports` 默认来源回退逻辑与 422 行为可测试
  - `/auth/me`、管理员路由守卫、普通用户访问 `/admin` 继续正确

- UX QA 交付：
  - 桌面正常宽度（约 1440）和更窄宽度（约 1152）各保留一轮 before/after 截图
  - 新增一份产品化 QA checklist，至少覆盖：
    - 首屏理解成本
    - composer 可达性
    - streaming 感知
    - evidence 干扰度
    - context 透明度是否适度
    - 管理员后台感是否污染普通聊天体验

## Assumptions
- 本轮以当前 `main` 为基线开发，但只做实验整改；是否合并回 `main` 由后续验收决定。
- 审查报告中的部分问题在 V1.1.7 已有局部缓解；本轮必须先做“问题仍是否存在”的核对，再整改仍成立的项。
- UI/user-facing 文案一律保持中文。
- 本轮默认最多使用 1 个只读子 agent；若实现期启用，只能用于审查材料比对或测试探查，不做并发写代码。

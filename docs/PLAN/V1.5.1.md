# V1.5.1 Session File Reading Workbench 执行冻结版

## Summary
- 本轮由 #1 实现 `session-file-reading-workbench`，把 AskPage 升级为支持上传、解析、分块、检索、引用的会话文件读取工作台。
- 复用现有 `files -> knowledge_items -> knowledge_chunks -> ask citations` 链路，不另建孤立 RAG，不改 #2 LightRAG 核心，不碰 #3 碳因子页。
- `frontend/src/styles/global.css` 当前聊天样式优化纳入本轮；`docs/碳核算专项/` 不纳入本轮提交。

## Key Changes
- 后端扩展 `files` 元数据和新增 `file_parse_results`，保存安全文件名、sha256、解析状态、parser、OCR、页/表/幻灯片数量、摘要、错误信息和 chunk_count。
- 新增 `backend/app/files/parser/` 管线：`ParserRegistry -> DoclingParser -> fallback parsers -> chunker`；Docling 优先，轻量 fallback 覆盖 txt/md/csv/html/pdf/docx/xlsx/pptx，图片/扫描 PDF 不可解析时明确失败。
- 上传安全固定为 allowlist、20MB 单文件、10 个 session 文件、200MB 用户空间、服务端生成 `<file_id>.<safe_ext>`，拒绝脚本、可执行、压缩包、宏 Office 和旧二进制 doc。
- Ask 接入新增 `session_file_search`，让 `attached_file_ids` 真实检索当前用户、当前 session、已 parsed/indexed 的上传 chunks；文件检索独立于 `knowledge_scope`。
- `private_upload` citation 增强 `file_id/page_number/sheet_name/slide_number/section_title`，回答中显示文件名和定位信息。
- AskPage 升级 paperclip、多文件上传、拖拽上传、附件 chip、状态轮询、解析摘要、本轮移除附件、发送时携带 parsed/indexed 的 `attached_file_ids`。

## Public APIs
- `POST /api/v1/files` 保留入参，响应增加解析状态、页/表/幻灯片数量、summary、knowledge_item_id。
- 新增 `GET /api/v1/files/{file_id}`，仅返回当前用户可见文件的上传、解析、索引、摘要和错误状态。
- `POST /api/v1/sessions/{id}/ask` 与 `/ask/stream` 请求体字段不变，但 `attached_file_ids` 必须生效。
- `AskCitation.source_type=private_upload` 增加可选定位字段：`file_id`、`page_number`、`sheet_name`、`slide_number`、`section_title`。

## Test Plan
- 后端覆盖上传安全、parser registry、Docling smoke、fallback parsers、chunk metadata、session_file_search、ask attached files、private_upload citations、跨用户隔离。
- fixtures 放 `backend/tests/fixtures/documents/`，至少覆盖 txt/md/csv/xlsx/docx/text-pdf；pptx/图片 smoke；扫描 PDF optional OCR smoke。
- 本地验证安装 `backend/requirements-optional.txt` 后运行 `scripts/document-parse-smoke.ps1`。
- 前端跑 `npm run typecheck && npm run build`，手工验证上传、解析、提问、文件 citation。
- 回归确保 public/private/mixed ask、#2 RAG、知识条目挂接、V1.4.x 碳核算不退化。

## Assumptions
- 不做供应商原生文件上传，不全文塞 prompt，不每轮重新解析文件。
- Docling 是主路线但仍为 optional dependency；CI 可 `importorskip`，本地验收必须安装跑通。
- #2 的 parser/chunker 适配只能作为未来可消费能力，不得覆盖 #1 本轮 `/api/v1/files`、AskPage、`attached_file_ids` 和 `session_file_search` 主线。

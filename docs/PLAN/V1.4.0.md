# CarbonRag V1.4.0 碳核算主线启动计划

## 轮次定位

V1.4.x 由 #1 首席团队负责，专攻碳核算能力增强。V1.3.x RAG / LightRAG 主线由 #2 团队负责，#1 只做审查与合并把关，不并行抢线。

本轮先冻结路线，不立即扩业务代码。

## 当前基础

现有 `calc-carbon` 已完成最小真实链路：

- 支持 `electricity_kwh`
- 支持 `natural_gas_m3`
- 支持 `diesel_l`
- 返回总排放量、分项排放量、因子引用和 `trace_id`
- 支持结果持久化
- 支持 session 关联
- 可被 `carbon_summary` 报告引用

## V1.4.x 目标

把碳核算从“最小三项活动数据计算”推进到“企业试用级核算模块”。

优先方向：

- 活动数据结构升级：支持更清晰的核算项目、周期、单位、备注和来源。
- 因子管理升级：支持因子版本、来源、启停、适用范围和审查状态。
- 计算解释增强：让每个分项都能追溯活动数据、公式、因子和引用来源。
- 结果复用增强：更好地接入 report、feedback、session 历史。
- 验证与测试增强：增加边界值、单位、缺失数据、因子缺失、权限隔离测试。

## 明确不做

- 不接管 V1.3.x RAG 主线。
- 不在 V1.4.0 直接引入完整行业核算平台。
- 不做自动从附件抽取活动数据。
- 不做正式审计认证结论。
- 不做 PDF/Word 正式报送模板。
- 不绕过 OpenSpec 与 PR 审查流程。

## 分支建议

- #1 碳核算开发：`t1/v1.4/<topic>`
- #2 RAG 开发：`t2/v1.3/<topic>`

## OpenSpec 要求

V1.4.x 正式开工前必须先建立 OpenSpec change，例如：

```text
openspec/changes/expand-carbon-accounting-foundation/
```

应影响 specs：

- `carbon-report-feedback`
- 如涉及 provider 或报告联动，再影响 `ai-runtime` / `conversation-memory`
- 如涉及部署配置，再影响 `devops-release`

## 验收标准

- README 与开发公告明确 V1.3.x / V1.4.x 分工。
- `main` 上默认中文首页可读、可跳转。
- V1.4.x 不打断 #2 的 V1.3.x 工作。
- 后续碳核算功能进入 OpenSpec propose -> apply -> archive 流程。

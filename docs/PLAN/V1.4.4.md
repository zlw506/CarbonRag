# CarbonRag V1.4.4 碳核算引擎源头对齐与 Factor-Driven Baseline

## Summary

V1.4.4 从现有三字段 `calc-carbon` 升级为因子驱动碳核算引擎基线，同时保留旧接口兼容。#2 继续负责 V1.3.x RAG / LightRAG 主线；#1 本轮只推进 V1.4.x 碳核算，不改 RAG 主线。

本轮以 `starrybodies/ghg-calculator` 作为计算引擎结构主参考，以 `EPFL-ENAC/co2-calculator` 作为模块化填报/因子模型参考。开源项目源码只放本地 `3rdparty/` 忽略目录，不提交、不复制 GPL 代码。官方数据口径以生态环境部/国家统计局电力因子、国家温室气体排放因子数据库为第一优先来源；CEADs 只作为宏观清单方法论参考。

## Key Changes

- 建立 OpenSpec change：`openspec/changes/carbon-engine-v2-factor-driven-baseline/`，影响 `carbon-report-feedback` spec；如实现触及报告联动或配置，再补 `conversation-memory` / `devops-release` delta。
- 新增研究文档：
  - `docs/research/carbon-accounting/open-source/ghg-calculator-engine-review.md`
  - `docs/research/carbon-accounting/open-source/epfl-co2-calculator-model-review.md`
  - `docs/research/carbon-accounting/source-data/upstream-source-decision.md`
- 后端碳核算结构升级为：
  - `ActivityItem / ActivityBatch`：统一活动数据输入。
  - `FactorRegistry`：按 scope、category、activity、region、year、source preference 选择因子。
  - `UnitConverter`：做最小单位归一与 conversion trace。
  - `CarbonCalculationEngine`：按 scope/category 路由到 Scope 1 / Scope 2 calculator。
  - `ResultSnapshot`：保存 factor snapshot、unit conversion trace、formula trace、warnings。
- `POST /api/v1/calc-carbon` 保留旧三字段兼容；若请求含 `activity_items[]`，走新引擎；旧字段内部转换成等价 activity item。
- 因子数据新增 `data/factors/carbon_v2_seed.json`，至少包含：
  - 官方 Scope 2 外购电力：2023 全国电力平均二氧化碳排放因子 `0.5306 kgCO2/kWh`，source 指向生态环境部/国家统计局公告。
  - 区域/省级电力因子可先放官方来源字段与结构，若未人工录全，不伪造数值。
  - natural gas / diesel / gasoline / LPG / coal 先用 `source_type=demo`，明确“不用于正式审计或正式盘查”。
- 第一阶段实现范围：
  - Scope 1 stationary combustion：natural gas、diesel、gasoline、LPG、coal。
  - Scope 1 mobile combustion：gasoline vehicle、diesel vehicle 最小版。
  - Scope 2 purchased electricity：location-based 完整实现。
  - Scope 2 market-based / green power：只保留字段与 warning，不完整计算；没有合同、绿证或交易凭证字段时不得把绿电算零。
  - Scope 3：只保留目录与类型预留，不计算。

## Public APIs / Types

`POST /api/v1/calc-carbon` 继续支持旧请求：

```json
{
  "session_id": "optional",
  "period_label": "2026-Q1",
  "electricity_kwh": 12000,
  "natural_gas_m3": 800,
  "diesel_l": 120
}
```

新增兼容请求：

```json
{
  "session_id": "optional",
  "organization_id": "org_demo",
  "facility_id": "facility_demo",
  "period_start": "2026-01-01",
  "period_end": "2026-03-31",
  "inventory_standard": "org_basic_v1",
  "activity_items": [
    {
      "scope": "scope2",
      "activity_category": "purchased_electricity",
      "activity_name": "electricity",
      "activity_value": 12000,
      "activity_unit": "kWh",
      "region": "CN",
      "year": 2023,
      "factor_preference": "official_latest"
    }
  ]
}
```

响应保持现有顶层字段，并新增快照字段：

```json
{
  "status": "ok",
  "trace_id": "calc-...",
  "total_emission_kgco2e": 1234.56,
  "breakdown": [],
  "formula_summary": "排放量 = 活动数据 × 排放因子",
  "citations": [],
  "factor_snapshot": [],
  "unit_conversion_trace": [],
  "formula_trace": [],
  "source_summary": [],
  "warnings": []
}
```

## Source Basis And Assumptions

- 官方电力因子来源采用生态环境部/国家统计局发布的 2023 年电力二氧化碳排放因子；国家能源局转载说明全国平均为 `0.5306 kgCO2/kWh`：[国家能源局转载](https://www.nea.gov.cn/20260116/b3867495909b4bf28f01c642d10e6345/c.html)，附件 PDF 可作为 seed 数据 citation 来源：[生态环境部 PDF](https://www.mee.gov.cn/xxgk2018/xxgk/xxgk01/202512/W020251231726284332528.pdf)。
- 国家温室气体排放因子数据库作为第一阶段因子库核心方向；第一版由生态环境部/国家统计局牵头上线，第二版扩容至 576 个因子：[第一版](https://www.mee.gov.cn/ywgz/ydqhbh/wsqtkz/202501/t20250110_1100380.shtml)，[第二版](https://www.mee.gov.cn/ywgz/ydqhbh/wsqtkz/202603/t20260301_1145117.shtml)。
- CEADs 只用于宏观清单方法论参考，不进入企业计算因子源。
- `3rdparty/ghg-calculator-main/` 与 `3rdparty/co2-calculator-dev/` 当前已存在且被 `.gitignore` 忽略；执行时可整理为本地 `3rdparty/carbon-study/` 便于阅读，但不得提交源码或 ZIP。
- `ghg-calculator` 本地 README 标称 MIT，但当前下载目录未发现 LICENSE 文件；执行时必须补一次上游许可证确认，未确认前只做结构参考。
- `EPFL-ENAC/co2-calculator` 本地 LICENSE 为 GPL-3.0；只学模型与 UX，不复制代码。

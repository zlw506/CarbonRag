# CarbonRag V1.1.8E Ask 页轻量化热修

## Summary
- 本轮只做 Ask 页的小范围体验热修，基线继续用当前实验分支，不扩业务、不改后端接口。
- 问题已经明确集中在 3 个前端责任区：
  - [frontend/src/pages/AskPage/index.tsx](/F:/Project/CarbonRag/CarbonRag/frontend/src/pages/AskPage/index.tsx)
  - [frontend/src/layouts/AppShell.tsx](/F:/Project/CarbonRag/CarbonRag/frontend/src/layouts/AppShell.tsx)
  - [frontend/src/styles/global.css](/F:/Project/CarbonRag/CarbonRag/frontend/src/styles/global.css)
- 本轮目标固定为：
  - 底部 composer 明显瘦身
  - Ask 首屏继续上提，聊天展示区更大
  - “标准工作台”入口移除
  - 用户入口从右上角头部挪到左下角头像浮钮
  - 上下文胶囊改成小圆形进度入口，不再独占一整行
  - 所有新增/修改文案继续保持中文

## Key Changes
- Ask 路由壳层继续专注化，但进一步去掉顶部占高：
  - Ask 页 focus mode 下，`AppShell` 不再渲染当前顶部 header 区。
  - Ask 页面主卡片直接顶到可用视口上边缘，让当前会话标题成为页面最上方主标题。
  - 右上角不再放当前用户菜单，避免与聊天层叠冲突。
  - 当前用户入口改为 Ask 页左下角悬浮头像按钮：
    - 默认只显示头像
    - 点击后以 popover / dropdown 展开用户名、角色标签、管理员入口（仅 admin）和退出登录
    - 普通用户仍不看到管理员入口

- “标准工作台”删除，顶部动作区只保留真正有用的入口：
  - Ask 页标题栏移除“标准工作台 / 专注模式”切换按钮。
  - 认为 Ask 页默认就是专注态，不再把“展开左栏”包装成一个高优先级产品动作。
  - 顶部右侧动作区只保留：
    - `参考资料`
    - 小型上下文圆形按钮
    - 可选的“更多设置”触发器
  - 当前 scope 标签从 composer 顶部移到标题右侧，作为紧凑状态标签显示，例如“公共政策 / 知识条目 / 混合”。

- Composer 重新压缩为单主行：
  - 底部 composer 去掉“当前会话：xxx”这一行。
  - 初始态主行只保留 3 个等高核心元素：
    - 左侧附件按钮
    - 中间输入框
    - 右侧发送按钮
  - 这 3 个控件视觉高度统一，输入框不再显得比图标按钮高一截。
  - `更多设置` 从 composer 顶部移到标题栏右上角，与 `参考资料` 并列或进入同一操作组。
  - scope、上传状态、知识条目挂接状态不再顶在 composer 上面；改成：
    - 标题右侧的当前 scope tag
    - 或 `更多设置` 弹层里的二级设置
    - 上传/发送中的运行状态只保留最小状态提示，不占用独立大行

- 上下文信息改成小圆形入口：
  - 现有 `chat-context-pill` 整块长条胶囊改成一个小圆形上下文进度按钮。
  - 默认显示方式固定为：
    - 一个小圆圈
    - 圆环进度代表上下文占用百分比
    - 不再默认显示“上下文 0%”文字和整条 meter
  - 点击圆形按钮后仍沿用当前详情弹层，展示：
    - 当前上下文占用（估算）
    - 最近消息数
    - 是否已压缩
    - 摘要更新时间
    - 本轮使用最近 N 轮 + 会话摘要 + M 条依据
  - 小圆按钮放在标题栏右上动作区，优先和“参考资料”放在同一视觉组，不再单独占一整行。

- 标题栏和消息区层级重新整理：
  - 当前会话标题继续作为主标题显示在 stream card 顶部。
  - 当前 scope tag 放到标题右侧。
  - 标题栏右侧放操作组：`更多设置`、`参考资料`、`上下文圆形按钮`。
  - 这样标题、状态、工具入口都收进同一条顶部带，不再把聊天区往下挤。
  - 左侧 session rail 保持现有可折叠行为，不额外新增一层“标准工作台”概念。

- 样式和层级修复：
  - 修掉当前 Ask 页 header / content / floating user trigger 的层级错位，保证用户头像浮钮不会再被聊天层盖住，也不会反过来挡住正文。
  - Ask 页左下角头像浮钮、右上角动作组、底部 composer 的 `z-index` 统一收口，避免互相遮挡。
  - composer 外边距、内边距再缩一档，减少底部占高，但不牺牲点击热区。
  - 输入框 placeholder、按钮、标签继续保持中文。

## Public APIs / Interfaces
- 不新增 API。
- 不改：
  - `POST /api/v1/sessions/{id}/ask`
  - `POST /api/v1/sessions/{id}/ask/stream`
- 只改 Ask 页前端行为契约：
  - Ask 页默认无顶部工作台 header
  - 当前用户入口改为左下角头像浮钮
  - scope 标签移到标题右侧
  - context 默认入口改为小圆形进度按钮
  - composer 默认只保留附件、输入框、发送三大主元素

  - 当前 scope 标签显示在标题右侧，不再占 composer 顶行

- Ask 页窄宽度：

## Assumptions
- 本轮继续在当前实验分支上做，不合并策略由后续验收再定。
- Ask 页默认长期保持专注态，因此“标准工作台”入口直接移除，而不是换个位置保留。
- 用户入口改为 Ask 页专用左下角头像浮钮；其他页面暂不改动其现有壳层表现。
- 上下文入口默认放在标题栏右上动作组中，用小圆环进度按钮实现；不采用右下角悬浮版本。
- 本轮只做前端热修，不改后端、不改流式协议。

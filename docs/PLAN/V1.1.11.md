# CarbonRag V1.1.11 会话列表滚动 + 左侧头像固定热修

## Summary
- 本轮只修左侧壳层布局，不动后端、不改现有问答/流式/主题逻辑。
- 根因已经明确：
  - 当前左侧 `Sider` 的“固定视口高度”只在 Ask 专注态上做得更强，其他工作台页仍可能跟随页面自然高度。
  - 左侧会话区和底部用户入口虽然已经分层，但会话列表的滚动责任还不够硬，导致会话多时会把底部头像挤出可见区，或滚动不稳定。
- V1.1.11 的目标固定为：
  - 左侧头像在所有带工作台壳层的页面都固定在最下方
  - 会话列表成为唯一可滚动区域，永远能滚到最底部并点中下方会话
  - 收起态只显示头像；展开态显示“左侧头像 + 右侧用户名”
  - 不改变现有会话切换、流式、主题、页面逻辑

## Key Changes
- 壳层高度和滚动责任收口：
  - `AppShell` 对所有工作台主页面统一启用固定视口高度布局，不只限于 Ask。
  - 左侧 `Sider` 统一拆成 4 段：
    - 品牌区
    - 主导航区
    - 会话区
    - 用户区
  - 只有“会话区 body”允许 `overflow-y: auto`；品牌区、导航区、会话头部、用户区都固定，不参与滚动。
  - 会话列表滚轮、触控板、触摸滚动都只作用在会话列表，不再把整个左栏或页面一起顶出视口。

- 左侧头像固定与显示规则：
  - 用户入口固定在左侧壳层最下方，所有使用 `AppShell` 的页面一致。
  - 收起态：
    - 只显示圆形头像按钮
    - 不显示用户名文字
  - 展开态：
    - 改成类似导航标题行的横向布局
    - 左侧头像
    - 右侧只显示用户名
  - 管理员入口、主题切换、退出登录等仍放在点击后的菜单/弹层里，不搬到常驻区域。

- 会话列表行为保持不变，但结构改成真正可滚：
  - 会话区继续保留“收起会话栏 / 新建对话 / 对话列表”结构
  - 展开态仍只显示会话标题，不恢复更新时间、消息数等碎片信息
  - 收起态行为不变，只隐藏会话列表，不显示两字缩略块
  - 不改变当前 `createSession / selectSession / refreshSessions / activeSessionId` 逻辑

- 作用范围：
  - Ask 页
  - 碳核算页
  - 报告页
  - 其他继续复用 `AppShell` 的工作台页
  - 目标是所有页面的左侧壳层行为一致，不再出现某页头像被挤走、某页会话区不滚的情况

## Public APIs / Interfaces
- 不新增后端 API。
- 不改：
  - `POST /api/v1/sessions/{id}/ask`
  - `POST /api/v1/sessions/{id}/ask/stream`
  - 当前主题配置、会话状态上下文、页面路由
- 前端行为契约更新为：
  - 左侧会话列表是唯一可滚动的工作台左栏区域
  - 左侧头像在所有工作台页固定到底部
  - 头像展开态显示为“头像 + 用户名”，收起态只显示头像

## Test Plan
- 左侧会话列表：
  - 制造足够多会话后，左侧会话列表可以稳定滚到最底部
  - 最下方会话可见、可点击、可切换
  - 滚动会话列表时不会把整个页面或整个左栏一起拖动
- 左侧头像：
  - Ask、碳核算、报告页都固定在左下区域
  - 展开壳层时显示头像 + 用户名
  - 收起壳层时只显示头像
  - 点击头像后主题、管理员入口、退出登录仍正常可用
- 页面回归：
  - Ask/碳核算/报告页现有主区布局不退化
  - 现有流式、thinking、主题切换、会话切换逻辑不退化
  - 收起/展开左栏时主内容区宽度行为保持现状，不引入新的空白问题
- 构建检查：
  - `npm.cmd run typecheck`
  - `npm.cmd run build`

## Assumptions
- 本轮是热修，不重构主题系统、不调整右侧抽屉、不改会话数据流。
- “展开态头像右侧介绍”为用户名本身，不额外常驻展示角色说明；角色信息继续放在弹层中。
- 修复范围以 `AppShell` 和左侧壳层样式为主，尽量不侵入各页面业务组件。

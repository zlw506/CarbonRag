# CarbonRag V1.1.9 全局会话栏上提 + 聊天主区放宽

## Summary
- 根因已经明确：当前是 `AppShell` 左侧导航 + 页面内 `SessionRail` 的双左栏结构。收起页面内会话栏后，全局壳层仍然占宽，所以聊天主区并没有真正放大。
- V1.1.9 的修复方向不是继续补 `SessionRail` 的缩略样式，而是把“收起对话栏 / 新建对话 / 对话列表”整体上提到全局左侧壳层，放到“报告生成”下面，所有主页面统一使用。
- 收起后，整个左侧壳层统一缩到图标态；不再显示两字缩略会话块。展开后才显示图标、文字和对话列表。这样 Ask 页空间会真实变大，碳核算页和报告页也统一。

## Key Changes
- 全局壳层改造：
  - `AppShell` 成为唯一的左侧工作台入口，左侧结构固定为：
    - 品牌区
    - 主导航：问答工作台 / 我的知识库 / 碳核算 / 报告生成
    - 会话区：收起按钮、新建对话、对话列表
    - 用户头像入口
  - 会话区固定放在“报告生成”下面，不再由 Ask / CarbonCalc / Report 页面自己渲染第二条会话栏。
  - 左侧壳层只有两种状态：
    - 展开：显示图标 + 文字 + 对话列表
    - 收起：只显示小图标，不显示对话列表、不显示两字缩略块
  - 默认收起宽度固定为小图标宽度，展开宽度固定为足够容纳标题的工作台宽度；不再存在“收起后还保留一条次级会话窄栏”的情况。

- 会话状态上提到壳层：
  - 新增一个前端工作台会话状态层，由壳层统一管理：
    - `sessions`
    - `activeSessionId`
    - `loadingSessions`
    - `createSession`
    - `selectSession`
    - `refreshSessions`
    - `sessionRailCollapsed`
  - Ask、碳核算、报告生成页不再各自 `listSessions/createSession/useResponsiveSessionRail`。
  - 三个页面统一从壳层上下文读取当前会话和会话切换能力。
  - 当前会话选择在主页面切换时保持一致，不再每个页面自己重选第一页第一条。

- 页面级会话栏彻底移除：
  - Ask 页删除页面内 `SessionRail` 列，只保留主聊天区和现有右侧依据抽屉逻辑。
  - 碳核算页删除页面内会话栏，只保留表单与结果主区。
  - 报告页删除页面内“会话选择 rail”，保留当前会话下的报告列表和报告工作区。
  - 结果是：
    - Ask 页收起左栏后，聊天主区直接变成单主列，宽度最大化
    - 碳核算页和报告页也不再有额外会话侧栏占宽

- 全局左栏的会话区交互：
  - 展开态：
    - 显示“新建对话”按钮，图标 + 文字
    - 显示会话列表，只保留标题，不显示“更新时间 / 消息数 / 附件数”
    - 当前会话高亮
  - 收起态：
    - 主导航全部只剩图标
    - 会话区不再渲染对话列表缩略块
    - 新建对话只保留一个加号图标
    - 会话区收起/展开按钮只保留图标
  - 不再尝试在收起态显示两字缩略，这条逻辑直接删除，避免再次出现裁切、竖排、假扩宽。

- Ask 页主区放宽：
  - 删除页面内左栏后，Ask 主布局改成单列主区。
  - 聊天消息区、标题区、底部 composer 继续保留当前 V1.1.8C/E/H 的行为，但以更宽的主区承载。
  - 右侧依据/状态仍保持抽屉，不重新变成常驻栏。
  - 助手消息卡片继续占满主区宽度，不再因历史双栏 CSS 残留保留左侧会话栏的空白。

- 统一到碳核算和报告页：
  - 碳核算页使用壳层会话区选择当前会话，不再在页面内部重复显示会话列表。
  - 报告页使用壳层会话区选择当前会话；页面内部只保留“当前会话报告列表”和报告配置/预览，不再有第二套聊天会话栏。
  - 这两页展开/收起左侧壳层时，行为和 Ask 完全一致。

## Public APIs / Interfaces
- 不新增后端 API。
- 不改：
  - `POST /api/v1/sessions/{id}/ask`
  - `POST /api/v1/sessions/{id}/ask/stream`
- 前端状态契约调整：
  - 会话选择从页面本地状态改为壳层共享状态
  - Ask / 碳核算 / 报告页共用同一套当前会话与会话切换逻辑
  - 收起态不再存在“会话两字缩略块”这个 UI 契约

## Test Plan
- Ask 页：
  - 展开左栏时，左侧导航下方可见“新建对话”和对话列表
  - 收起左栏后，对话列表完全消失，只剩小图标导航和小图标按钮
  - 收起后聊天主区宽度明显变大，不再保留额外左侧空栏
  - 右侧依据按钮、更多设置、composer、streaming/thinking、滚动归属不退化

- 碳核算页：
  - 不再有页面内第二条会话栏
  - 会话切换统一走左侧全局会话区
  - 收起左栏后主内容区真实变宽
  - 新建对话、切换对话正常

- 报告页：
  - 不再有页面内第二条聊天会话栏
  - 左侧全局会话区可切换当前会话
  - “当前会话报告列表”继续正常，只展示报告，不混入聊天会话 rail
  - 收起左栏后主内容区真实变宽

- 回归：
  - `npm.cmd run typecheck`
  - `npm.cmd run build`
  - 暗色模式下左侧壳层、会话区、主内容区不出现断层
  - 普通用户/管理员都验证一轮，确保宽度差异不是角色导致，而是布局已统一

## Assumptions
- 本轮继续在当前实验分支推进，不合并到 `main`，除非后续单独要求发布。
- 本轮统一的是“聊天会话栏”逻辑，不包括报告页里的“报告列表”。
- 收起态直接隐藏对话列表，不再保留两字缩略块；这是为了解决空间浪费和缩略异常的根问题。
- 会话共享状态优先放在前端壳层上下文中实现，不引入新的后端接口或路由形态。

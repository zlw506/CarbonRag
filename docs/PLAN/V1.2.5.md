# CarbonRag V1.2.5 统一开放协作指南冻结轮

## Summary

- 本轮不做业务功能、不改 UI、不修普通 bug，只冻结所有席位公开透明遵守的协作指南。
- 发布口径调整为：当前本地所有未忽略、已暂存/已跟踪/明确属于协作基线的文件都纳入云端，包括 V1.1.x 计划文档与 UX 审查文档。
- 完成后把 `t1/v1.2/openspec-governance-baseline` 合入 `main`，让 `main` 成为 #2/#3 fork、clone、本地测试和 PR 的共同入口。

## Key Changes

- 新增统一协作 runbook：
  - `docs/governance/OPEN_COLLABORATION_GUIDE.md`
  - `docs/governance/SEAT_ONBOARDING_RUNBOOK.md`
  - `docs/governance/OPENSPEC_CODEX_WORKFLOW_RUNBOOK.md`
  - `docs/governance/PR_REVIEW_RUNBOOK.md`
  - `docs/governance/TRACKED_COLLABORATION_ASSET_INVENTORY.md`
  - `docs/governance/OPEN_COLLABORATION_GUIDE_TEST_REPORT.md`
- 新增 OpenSpec change：`formalize-open-collaboration-guide`，只覆盖治理协作能力，不改业务行为。
- 更新 README、开发启动文档、部署文档、Git 工作流与 release flow，让新席位从云端仓库即可完成本地测试。
- 明确忽略文件不是“原版缺失”：`.env`、依赖目录、SQLite、uploads、`.spec-gen/`、`3rdparty/spec-gen/` 都由模板、脚本或本地命令重建。
- 安装并测试 GitHub CLI；安装并核验 VS Code GitHub Pull Requests 扩展。

## Public Contracts

- `main`：稳定源码基线、当前公网默认发布基线、所有席位共同入口。
- `t1/v1.2/<topic>`：#1 upstream 开发分支。
- `t2/v1.2/<topic>`：#2/#3 fork-and-PR 开发分支。
- `Git-ys1`：`main` 最终管理员、PR 最终审查人、CODEOWNERS fallback owner。
- 新功能默认必须先有 `openspec/changes/<change-id>`；docs-only 和 emergency hotfix 例外必须在 PR 中说明。

## Verification

- `openspec validate formalize-open-collaboration-guide --strict`
- `openspec validate --all`
- `git status --ignored -sb`
- `git ls-files`
- `git ls-files -o --exclude-standard`
- `git check-ignore -v .spec-gen/test.txt`
- `git check-ignore -v 3rdparty/spec-gen/test.txt`
- `git check-ignore -v .env`
- `gh --version`
- `gh auth status`
- `gh pr list`
- `code.cmd --install-extension GitHub.vscode-pull-request-github --force`
- 后端 pytest、前端 typecheck/build 作为合入 main 前回归验证。

## Assumptions

- V1.1.x 计划文档与 UX 审查文档可公开进仓库。
- `backend/app/ai_runtime/tools/enterprise_retrieve_stub.py` 不含密钥、本地路径或临时垃圾，因此按未忽略资产纳入发布。
- GitHub CLI 需要用户执行 `gh auth login` 后才能完成 PR 列表和 review 操作。
- 本轮不修复因环境或既有业务测试失败暴露的问题；只记录阻塞项。

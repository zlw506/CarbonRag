# CarbonRag V1.1.13 可靠性重连 + 通用设置/API 配置主线

## Summary
- 本轮只做两条主线：`问答首轮失败后的可靠性重连/恢复`，以及 `通用设置 + provider/model 可配置化`。
- 已锁定的关键决策：
  - 可重试错误固定包含：连接错误、连接/读取超时、上游 `408/409/429/5xx`、以及首个有效内容 token 前断流。
  - 自动重连最多 `5` 次，退避固定为 `0.8s -> 1.6s -> 3.2s -> 5s -> 8s`，每次附少量 jitter。
  - mid-stream 断流优先走 `resume`，不是整轮重开。
  - provider 凭据本轮两种都做：
    - `仅当前设备使用`：浏览器本地保存，敏感 key 只放本地私有存储
    - `保存到账号设置`：后端加密保存
  - 当前“更多主题”升级为“通用设置”，不再把主题、聊天偏好、provider/model 配置散在不同位置。

## Key Changes
- **主线 A：问答可靠性、流式恢复、标题补偿**
  - 在 ask/stream 链路新增 `request_group_id + event_seq + resume_cursor` 语义；同一条用户消息整条重连链路保持同一个 `request_group_id`。
  - 后端新增一个进程内 `active stream registry`：
    - 保存当前流的消息 ids、累计 answer/thinking、最近事件序号、重试次数、是否已收到首 token
    - 客户端断线后可按 `request_group_id + resume_cursor` 继续接回
    - 只有在首 token 前失败才走“首轮重连”；已开始稳定输出后断流优先走“恢复当前流”
  - ask/stream 的状态机扩成：
    - `connecting`
    - `thinking`
    - `reconnecting`
    - `streaming`
    - `done`
    - `failed`
  - `status` 事件不再只带状态名，还要带：
    - `request_group_id`
    - `attempt`
    - `max_attempts`
    - `recovered`
    - `resume_supported`
  - 前端 Ask 页固定按这套状态显示中文用户文案：
    - 正在连接模型…
    - 连接中断，正在重试（x/5）…
    - 第 x 次重连成功
    - 已恢复，继续生成
    - 本次未能连接到模型，请稍后重试或检查模型设置
  - SDK/transport 分层固定为两层：
    - provider SDK 层暴露 `timeout/max_retries`
    - CarbonRag runtime 层再包“会话级重连 + resume + 消息状态补偿”
  - 标题策略改为“成功后补偿生成”，不再在首条用户消息发出时永久落标题：
    - 新建会话只写临时标题 `新对话 YYYY-MM-DD HH:mm`
    - 只有 assistant 首次成功完成后才生成正式标题
    - 正式标题基于前 `1~3` 条用户消息的摘要生成
    - 前三条用户消息内允许最多持续重算；第 3 条之后冻结
    - 首轮失败、后续重连成功时必须补跑标题生成
  - 标题生成不再依赖远端模型；本轮固定使用本地摘要规则，避免标题补偿再被 provider 故障拖垮。

- **主线 B：通用设置 + provider/model 配置**
  - 新增统一设置页 `/settings`，头像菜单改为：
    - 通用设置
    - 管理后台
    - 退出登录
  - 通用设置固定分 5 区：
    - 外观
    - 聊天
    - 模型与提供方
    - 数据与隐私
    - 高级
  - 外观区承接现有主题系统：
    - 主题模式：浅色 / 暗色 / 跟随系统
    - 主题预设：保留现有主题卡
    - 气泡密度：舒适 / 紧凑
    - 字体大小：默认 / 大号
    - 侧栏默认：展开 / 收起
  - 聊天区固定提供：
    - `Enter 发送 / Ctrl+Enter 发送`
    - 思考过程默认展开
    - 依据面板默认显示
    - 上下文调试信息默认显示
    - 新会话自动标题开关
    - 重连提示显示方式
  - 模型与提供方区作为本轮重点：
    - 内置 `CarbonRag 默认云端`
    - `自定义 OpenAI-compatible`
    - `Ollama`
    - `OpenAI`
    - `Anthropic`
    - `Gemini`
    - `DeepSeek`
  - 先完成 `默认云端 / 自定义 OpenAI-compatible / Ollama` 全流程可用，再补齐 OpenAI / Anthropic / Gemini / DeepSeek。
  - provider 配置固定采用“可见配置 + 敏感凭据分离”：
    - 非敏感设置和当前生效 provider 保存在账号设置
    - 本地私有 provider 的敏感 key 只放浏览器本地私有存储
    - 账号保存的敏感 key 后端加密落库，不回传明文

- **存储与运行时分层**
  - 运行时数据库新增两张表：
    - `user_settings`
    - `user_provider_profiles`
  - `user_settings` 只存用户级偏好和当前生效 provider ref：
    - `appearance_json`
    - `chat_json`
    - `data_privacy_json`
    - `advanced_json`
    - `active_provider_ref`
  - `user_provider_profiles` 只存账号级 provider 资料：
    - `profile_id`
    - `owner_user_id`
    - `provider_type`
    - `display_name`
    - `base_url`
    - `model_name`
    - `config_json`
    - `api_key_encrypted`
  - `active_provider_ref` 用统一引用语义，固定允许：
    - `builtin:carbonrag-cloud`
    - `account:{profile_id}`
    - `local:{profile_id}`
  - `local:*` profile 不进数据库，只在前端本地保存；当前 ask/report 请求携带 `provider_override`
  - `builtin:*` 和 `account:*` 由后端 `provider resolver` 解析；Ask、stream、report、memory compaction 一律走同一解析器，不再只吃全局 env 配置

## Public APIs / Interfaces
- **新增设置接口**
  - `GET /api/v1/settings`
  - `PATCH /api/v1/settings`
  - `GET /api/v1/settings/providers`
  - `POST /api/v1/settings/providers`
  - `PATCH /api/v1/settings/providers/{profile_id}`
  - `DELETE /api/v1/settings/providers/{profile_id}`
  - `POST /api/v1/settings/providers/test`
  - `POST /api/v1/settings/providers/discover-models`
- **新增 settings/provider 关键类型**
  - `ThemeMode = light | dark | system`
  - `ProviderType = carbonrag_cloud | openai_compatible | ollama | openai | anthropic | gemini | deepseek`
  - `CredentialStorageMode = local_only | account`
  - `ActiveProviderRef = builtin:* | account:* | local:*`
- **现有 ask / stream / report 请求扩展**
  - `AskRequest` 与 `CreateReportRequest` 增加可选：
    - `request_group_id?: string`
    - `resume_cursor?: number`
    - `provider_override?: LocalProviderOverride`
  - `provider_override` 只在当前激活 provider 属于 `local:*` 时发送
- **SSE 事件扩展**
  - 事件名保持不变：`message_start / status / thinking_delta / answer_delta / metadata / done / error`
  - `status` 事件增加：
    - `request_group_id`
    - `attempt`
    - `max_attempts`
    - `recovered`
    - `resume_supported`
  - `metadata/done` 增加：
    - `request_group_id`
    - `title_updated?: boolean`
    - `provider_ref`
- **模型发现固定规则**
  - `CarbonRag 默认云端`：返回当前服务允许的默认模型列表，至少包含当前已配置模型
  - `OpenAI / DeepSeek / 自定义 OpenAI-compatible`：`GET /models`
  - `Anthropic`：Models API
  - `Gemini`：models.list / models endpoint
  - `Ollama`：`GET /api/tags`

## Test Plan
- **重连与流式恢复**
  - 首 token 前网络错误 / 超时 / `429` / `5xx`：自动重连，最多 5 次
  - 用户能看到第几次重连
  - 第 2~5 次成功后消息继续完成，不重复生成、不丢已有气泡
  - mid-stream 断流后优先 resume；恢复成功时不新增第二条 assistant 消息
  - 5 次全部失败后进入明确 `failed` 状态并出现重试入口
  - 失败消息不会形成“空白但成功”的假完成
- **标题补偿**
  - 首轮失败、后续成功时标题会补跑生成
  - 标题基于前三条用户消息摘要，不再只取第一句
  - 第 3 条用户消息后标题冻结
- **设置与 provider**
  - 通用设置页可正常读取/保存 5 个分区
  - `默认云端 / 自定义 OpenAI-compatible / Ollama` 可切换
  - 至少一个 provider 能自动发现模型
  - 激活 provider/model 后，ask/stream 的实际请求使用新的 provider 配置
  - 本地保存与账号保存两种模式都可用，后端不回传明文 key
- **页面回归**
  - 头像菜单改成“通用设置 / 管理后台 / 退出登录”
  - 现有主题系统迁入设置页后不退化
  - Ask、知识库、碳核算、报告页不因设置改造退化
- **后端/前端验证**
  - 新增后端单元与路由测试覆盖：retry 分类、resume、title compensation、settings CRUD、provider discovery/test
  - 新增前端 smoke：首轮失败重连提示、settings 切换 provider、生效 ask/stream、刷新后设置保留
  - 继续通过：
    - `pytest`
    - `npm.cmd run typecheck`
    - `npm.cmd run build`

## Assumptions
- 这轮两种凭据保存方式都做：
  - `local_only` 只在浏览器本地私有存储保存敏感 key
  - `account` 使用后端加密保存，依赖新增环境变量 `SETTINGS_ENCRYPTION_KEY`
- 当前云端仍按单进程/单实例工作台部署假设实现 `active stream registry`；若进程重启，正在进行中的流可回退为失败并给用户显式重试入口，不做跨进程恢复。
- provider 解析器将替代当前只读全局 env 的 chat provider 获取方式；Ask、stream、report、memory compaction 统一走这套解析器。
- 标题生成本轮固定为本地摘要规则，不额外依赖远端模型调用。
- V1.1.13 完成并通过本地验证后，按快进方式合入 `main`。

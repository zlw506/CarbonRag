# CarbonRag v0.1.9C 前后端部署配置计划

## 目标
- 补齐 VPS 后端生产部署配置
- 补齐 Netlify 前端部署配置
- 固化生产环境变量、systemd 与验证口径
- 保持 ask / calc-carbon / feedback / session 工作台可直接部署

## 本轮范围
- 不扩业务能力
- 不改数据库口径
- 不把后端塞进 Netlify Functions
- 不要求用户手动改源码

## 本轮关键决策
- 生产运行时持久化统一切到 PostgreSQL
- 本地开发与测试继续保留 SQLite fallback
- 生产前端统一使用 `VITE_API_BASE_URL=/api`
- 前端服务层请求 `/v1/...`
- Netlify 代理 `/api/* -> http://8.141.111.33/api/:splat`

## 交付物
- `netlify.toml`
- 更新后的 `.env.example`
- 更新后的 `frontend/.env.example`
- `docs/deploy/VPS_BACKEND_DEPLOY.md`
- `docs/deploy/NETLIFY_FRONTEND.md`
- `docs/deploy/carbonrag.service`
- PostgreSQL / SQLite 双运行时存储切换代码

## 验收
- 本地 SQLite 模式不退化
- 生产配置支持 `DATABASE_URL` 与 `UPLOAD_DIR`
- ask / files / calc-carbon / feedback 可在 PostgreSQL 模式下持久化
- 前端生产构建不再包含本地 `127.0.0.1` API 地址
- 文档能直接指导 VPS 与 Netlify 配置

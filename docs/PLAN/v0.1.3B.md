# CarbonRag v0.1.3B 静态快照研究轮实施计划

## Summary
- 本轮正式定级为 `v0.1.3B = 静态快照研究轮`。目标是研究 `claw-code` 的架构分层并把结论沉淀进 CarbonRag 仓库，不做接入、不做替换、不做业务改造。
- 当前本地事实已经明确：`3rdparty/` 下有 `main`、`fix/plugin-loading-parity`、`fix/skill-invoke`、`fix/ui-parity` 四份静态目录快照和 zip 包，但都不含 `.git`；因此本轮只做目录/文件级对照，不做 branch history、commit history 或 merge-base 分析。
- 施工基线从当前最新工程状态 `feature/stable-bootstrap` 续开，建议新分支名固定为 `feature/claw-code-study-baseline`。提交纪律固定为：先 `checkpoint: before v0.1.3b`，最终提交 `v0.1.3b: complete static snapshot study baseline for claw-code`。

## Key Changes
- `3rdparty/` 本地学习区按统一形态整理为：
  - `3rdparty/clone-zip/` 保留四个 zip 原包
  - `3rdparty/claw-code-study/main_snapshot/`
  - `3rdparty/claw-code-study/fix_plugin_loading_parity_snapshot/`
  - `3rdparty/claw-code-study/fix_skill_invoke_snapshot/`
  - `3rdparty/claw-code-study/fix_ui_parity_snapshot/`
  - 目录要扁平化到真实项目根，直接能看到 `README.md`、`CLAW.md`、`PARITY.md`、`rust/`、`src/`、`tests/`
- 仓库级忽略规则收紧为本地学习资产全忽略，而不是只忽略单一路径：
  - `.gitignore` 新增 `/3rdparty/*`
  - `.gitignore` 新增 `!/3rdparty/.gitkeep`
  - 提交 `3rdparty/.gitkeep`
  - 不提交任何 zip、静态快照、Rust 构建产物或外部仓源码
- Rust 基线按本机可行路径固定：
  - 这台机器没有 `winget` / `choco`，后续实施时使用官方 `rustup-init.exe` 直装 stable toolchain
  - 安装后先验 `rustup --version`、`rustc --version`、`cargo --version`
  - 只对 `main_snapshot/rust` 跑一次 `cargo build --release`、`cargo fmt --check`、`cargo clippy --workspace --all-targets -- -D warnings`、`cargo test --workspace`
  - 这四条命令的结果必须记入 `05_LOCAL_SETUP_AND_NOTES.md`；本轮不强制四个快照都构建
- 新增并补齐 `docs/research/claw-code/` 六份文档，全部非空：
  - `00_STUDY_SCOPE.md`：写清研究目的、边界、固定阅读顺序、静态快照限制、明确“不接入 CarbonRag 运行时”
  - `01_RUST_CORE_MAP.md`：按本地真实 workspace 命名写 `api`、`runtime`、`tools`、`commands`、`plugins`、`compat-harness`、`claw-cli`，并明确注明 README/任务书里把 `api` 概念性称为 `api-client`
  - `02_SRC_SURFACE_MAP.md`：只研究 `src/assistant`、`services`、`skills`、`hooks`、`plugins`、`state`、`memdir`、`cli` 的表面职责，不延伸成 CarbonRag 改造方案
  - `03_PARITY_GAP_MAP.md`：把 `PARITY.md` 的 gap 重新翻译成 CarbonRag 语境，至少覆盖 `plugins`、`hooks`、`skills`、`cli`、`assistant orchestration`、`services`、`memory/session integration`
  - `04_CARBONRAG_ADOPTION_MATRIX.md`：只写 `必须借鉴 / 暂缓借鉴 / 明确不借鉴` 三栏和理由，不写实现；明确不借鉴项至少包括“直接拷贝 claw-code 源码”“把 coding-agent CLI 直接当 CarbonRag 产品壳”“把静态研究包装成已集成”
  - `05_LOCAL_SETUP_AND_NOTES.md`：必须写死“当前样本为静态目录快照、无 `.git` 元数据”，并记录样本来源、可做/不可做、Rust 结果、后续若要补 git 元数据该怎么做
- 分支对照方式固定为目录级 diff，不再假装是 git 级研究：
  - 对照顺序固定 `main -> fix/plugin-loading-parity -> fix/skill-invoke -> fix/ui-parity`
  - 优先看本地已暴露的差异热点：`rust/crates/commands`、`rust/crates/tools`、`rust/crates/plugins`、`rust/crates/claw-cli`、`rust/crates/runtime/src/skills.rs`，以及 `README.md` / `PARITY.md`
- 仓库文档入口同步补一笔：
  - 更新 `README.md` 的文档索引，加入 `docs/research/claw-code/`
  - 不改 CarbonRag 后端、前端、provider、API 边界草案和任何运行时逻辑

## Public Interfaces / Contracts
- 本轮不新增应用 API、schema、后端路由或前端交互协议。
- 新增的仓库级契约只有两类：
  - `3rdparty/` 是本地学习区，默认全部忽略，唯一允许追踪的是 `.gitkeep`
  - `docs/research/claw-code/` 是 v0.1.3B 的正式研究输出面，后续学习必须在这里累积，而不是只留本地笔记

## Test Plan
- `git status -sb` 在补完 `.gitignore` 后不再出现 `?? 3rdparty/`
- `git check-ignore -v` 能确认四份快照和 zip 包被忽略，而 `3rdparty/.gitkeep` 不被忽略
- 四份快照都完成真实根目录检查：存在 `README.md`、`CLAW.md`、`PARITY.md`、`rust/`、`src/`、`tests/`，并把 `.git` 缺失情况写进 `05_LOCAL_SETUP_AND_NOTES.md`
- `git diff --no-index --stat` 至少完成三组对照：`main vs plugin-loading`、`main vs skill-invoke`、`main vs ui-parity`，并把热点结论写入 `01/02/03/04` 对应文档
- Rust toolchain 装好后，`cargo` 四条命令至少在 `main_snapshot/rust` 执行一次；成功或失败都必须留痕到 `05_LOCAL_SETUP_AND_NOTES.md`
- 六份研究文档全部非空，其中 `01/02/03/04/05` 必须有可读的第一版有效内容
- 提交历史清晰，包含 checkpoint 和最终版本提交两笔

## Assumptions
- 本轮按你拍板后的 `v0.1.3B` 验收，不要求完整 git clone/worktree，也不做 commit 演化分析。
- 新工作从 `feature/stable-bootstrap` 续开，而不是从落后的 `dev` 重新起分支。
- 本地静态快照是本轮唯一研究样本；若后续要做真正分支历史研究，另开 `v0.1.3C` 类补正轮。
- Rust 安装默认走官方 `rustup-init.exe`，因为本机没有 `winget` / `choco`，但官方分发地址当前可达。

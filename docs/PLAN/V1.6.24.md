# V1.6.24 RAG-Pro Usability and Pipeline Hardening

## Summary
- 目标：把现有 RAG-Pro spine 从“能跑但分散”收敛成“上传后可一键入库、可验收、可解释失败”的产品闭环。
- 当前核验：`README.md` 已更新到 `V1.6.24`；KB 上传、parse、chunk、index、search、test-qa、eval 已有雏形；`/rag/retrieve` 已降级为 admin-only legacy；GitNexus 已重建，可用于本轮影响面复核。
- 本轮不做新算法、不做知识图谱、不做碳核算；继续允许直接迁移/改写 RAG-Pro 核心结构与逻辑。

## Key Changes
- 更新入口文档：`README.md` 顶部改为 `V1.6.24`，`开发公告.md` 和 `docs/PLAN/V1.6.24.md` 写清正式验收路线：`/rag/search` 只测检索，`/rag/answer` 正式生成，`/rag/test-qa` 工作台测试，`/rag/eval/run` 验收评分，`/rag/retrieve` 为 legacy。
- 新增 OpenSpec change：`rag-pro-usability-and-pipeline-hardening`，影响 `knowledge-rag`、`governance`、`devops-release`。
- 新增一键入库接口：`POST /api/v1/kb/{kb_id}/documents/{doc_id}/run-pipeline`，按 `parse -> chunk -> index -> search smoke -> eval smoke` 顺序执行，任一阶段失败即停止并返回 `failed_stage/error_message/warnings`。
- 新增批量入库接口：`POST /api/v1/kb/{kb_id}/documents/run-pipeline-batch`，默认处理当前 KB 下未 indexed 或 failed 的文档，也支持传 `doc_ids` 限定范围。
- Pipeline 返回固定字段：`doc_id/parse_status/chunk_status/index_status/chunk_count/indexed_chunk_count/vector_runtime/degraded/search_smoke_passed/eval_passed/failed_stage/warnings`。
- Search smoke 默认用首个非空 chunk 片段或文档标题生成内部查询，要求至少命中同一 `doc_id`；eval smoke 仅在默认 `gold_questions.json` 可用时运行，否则返回 `eval_passed=null` 和 `eval_not_configured` warning。
- 工作台默认主按钮改为“一键入库验收”，手工“解析文档 / 生成片段 / 写入向量库”保留为高级操作；失败文档显示“重试失败阶段”，实际调用同一个 pipeline 并从当前失败阶段继续。
- `/api/v1/rag/retrieve` 改为 admin-only legacy endpoint；`RagLabPage` 保持“Legacy RAG 实验台”且普通用户导航隐藏；文档和 UI 不再把 `langchain_rag_*` 写成主路线。
- 正式工具名固定为 `rag_pro_search`、`rag_pro_answer`；旧 `langchain_rag_search/answer` 只保留兼容和回归测试。
- AskPage 增加 RAG 证明面板，展示 `kb_id/知识库名/rag_mode/provider/model/vector_runtime/degraded/dense_count/sparse_count/merged_count/rerank_applied/citation_count/selected_chunks/warnings`。
- AskPage 对 `degraded=true`、`vector_runtime=memory_dev`、`dense_count=0`、`citation_count=0`、`hybrid_rerank` 下未 rerank 的情况显示红色或橙色风险提示。
- Workbench eval 从工具升级为门禁：一键入库后自动尝试默认 eval；“运行验收评分”面板展示未命中问题、命中 chunk、expected keywords、actual topK snippets、跨库泄漏、向量失败、rerank 失败。

## Public APIs / Types
- `POST /api/v1/kb/{kb_id}/documents/{doc_id}/run-pipeline`：单文档一键入库验收。
- `POST /api/v1/kb/{kb_id}/documents/run-pipeline-batch`：批量入库验收，响应包含 `total_count/succeeded_count/failed_count/results[]`。
- `RagDocument` 状态继续使用现有 `parse_status/chunk_status/index_status/parse_progress/chunk_progress/error_stage/error_message`，不新增第三套状态。
- `/api/v1/rag/search`、`/api/v1/rag/answer`、`/api/v1/rag/test-qa`、`/api/v1/rag/eval/run` 是唯一验收入口。
- `/api/v1/rag/retrieve` 保留但标记 legacy/admin-only，不参与 V1.6.x RAG-Pro 验收。

## Test Plan
- 后端新增测试：单文档 pipeline 成功、parse 失败停止、index 失败返回 `failed_stage=index`、批量 pipeline 汇总、search smoke 同 doc 命中、eval smoke 无配置时 warning。
- 后端新增测试：`/rag/retrieve` 普通用户不可访问、admin 可访问；`rag_pro_search/rag_pro_answer` 为正式工具名；旧 `langchain_rag_*` 仍兼容。
- 后端新增验收：同一 KB 的 Workbench Test QA 与 AskPage 对“青木制造 2025 年第一季度合计外购电力是多少？”都命中 `217,650 kWh`，且 `trace.kb_id` 一致。
- 文件格式 smoke：`md/txt/pdf/docx/xlsx` 至少验证上传、解析、chunk_count、indexed_chunk_count、可检索、可回答、citation 是否带文件名/页码/sheet。
- 前端验证：工作台主按钮和状态文案中文化；AskPage RAG 证明面板可展开；风险 trace 明确标红；Legacy RAG 仅 admin 可见。
- 回归命令：`openspec validate rag-pro-usability-and-pipeline-hardening --strict`、`openspec validate --all`、`cd backend && .\.conda\python.exe -m pytest`、`cd frontend && npm.cmd run typecheck && npm.cmd run build`、`git diff --check`。
- GitNexus：实施前重建 CarbonRag 索引；完成后用 `gitnexus detect_changes` 复核 RAG/API/frontend 影响面。

## Assumptions
- Docker Milvus、BGE-M3、reranker 是真实验收前置；`memory_dev` 或 `degraded=true` 不能算通过。
- Pipeline 本轮先做同步执行，不引入 Celery/RabbitMQ/MinIO；长任务队列化留给后续 ragPdfSystem 企业增强阶段。
- `docs/碳核算专项/` 继续不纳入本轮提交。
- RAG-Pro 仍是主复现对象，允许直接迁移/改写其核心结构与逻辑，但不得提交 `3rdparty` 源码树。

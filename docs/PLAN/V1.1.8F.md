# CarbonRag V1.1.8F 全局壳层收口 + Ask 轻量化热修

## Summary
- 先按你的要求做一次独立 `checkpoint` 提交，冻结当前已完成的 V1.1.8B/C/E 改动；再单独落 V1.1.8F 热修。
- 本轮只改前端壳层和 Ask 页，不改后端、不改流式协议。
- 目标收口为 3 件事：
  - Ask 左侧会话区更干净：不再显示会话栏标题、辅助说明、会话卡片元信息
  - 用户入口改成全站统一的左侧常驻头像入口，不再只在 Ask 特化，也不再放右上角
  - Ask composer 和上下文入口继续压缩：输入栏更薄、顶部动作更简、小圆形上下文入口保留

## Key Changes
- 提交顺序固定：
  - 先提交当前工作树为独立 checkpoint，不混入 V1.1.8F
  - 再在同一实验分支上提交 V1.1.8F 热修
- 全局壳层统一：
  - `AppShell` 去掉所有页面右上角用户菜单
  - 用户入口改成左侧 `Sider` 内的常驻紧凑头像入口，并在所有标签页一致显示
  - 位置按你要求改为左侧中段/导航区下方的常驻入口，不再放左下角浮动位
  - 点击头像后再展开用户名、角色、管理员入口、退出登录
- Ask 左侧会话区继续精简：
  - 会话栏卡片头不再显示“会话/会话列表”文字
  - “新建会话”按钮改成纯加号按钮，不再显示文字
  - 删除“中间消息区是主视觉区；这里只保留会话切换。”这类说明文案
  - 会话卡片只保留标题，不再显示“更新于… / …条消息 / …个附件 / …个知识条目”
- Ask 顶部与 composer 收口：
  - 保留当前标题右侧 scope 标签、小圆形上下文按钮、`参考资料`、`更多设置`
  - 底部 composer 继续保持单主行：附件按钮、输入框、发送按钮
  - 不恢复“当前会话：xxx”或其他额外说明行
  - 不改变当前 streaming / thinking / 滚动归属实现
- 样式统一：
  - 全局 `Sider` 内新增用户入口样式，兼容展开/收起两种导航态
  - Ask 会话栏头部按钮、纯标题会话卡片、composer 高度和间距同步收紧
  - 处理全局 header / sidebar / message 层级，避免用户入口与主内容再发生遮挡

## Public APIs / Interfaces
- 不新增 API
- 不改：
  - `POST /api/v1/sessions/{id}/ask`
  - `POST /api/v1/sessions/{id}/ask/stream`
- 前端行为契约更新为：
  - 用户入口全站统一为左侧常驻头像入口
  - Ask 会话列表卡片只显示标题
  - Ask 会话栏头部不再显示标题文字与“新建会话”文字
  - Ask composer 保持单主行，小圆形上下文入口继续保留

## Test Plan
- 全站壳层回归：
  - Ask、我的知识库、碳核算、报告页都不再显示右上角用户菜单
  - 左侧常驻头像入口在所有页面都可见且可点
  - admin 仍能在头像菜单内进入后台，普通用户看不到后台入口
- Ask 页回归：
  - 会话栏头部只剩折叠按钮和加号新建按钮
  - 不再显示“会话列表”标题和说明文案
  - 会话卡片只显示标题
  - composer 仍为单主行，流式、thinking、滚动归属不退化
  - 小圆形上下文按钮仍可点击展开详情
- 构建检查：
  - `npm.cmd run typecheck`
  - `npm.cmd run build`

## Assumptions
- V1.1.8F 继续在当前实验分支推进，不合并到 `main`
- 本轮不改后端、不补新文档，除非需要新增一份简短计划记录来配合 checkpoint
- “左中间”按实现收口为：用户入口放进全局左侧导航区下方的常驻位置，而不是右上角，也不是 Ask 专用左下角浮动位

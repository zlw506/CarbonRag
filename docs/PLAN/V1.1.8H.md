# CarbonRag V1.1.8H 会话缩略回退修正 + 多页会话栏统一

## Summary
- 本轮继续在当前实验分支上做前端热修，不改后端、不改接口。
- 已定位的问题有两层：
  - Ask 页收起态会话栏仍沿用完整卡片壳层和过窄列宽，导致两字缩略被裁切、主聊天区没有获得足够的可见扩展。
  - 碳核算页和报告页还在用旧版会话列表实现，没有继承 Ask 页这轮之后的“标题优先、可折叠、无碎片元信息”逻辑。
- 本轮目标固定为：
  - Ask 页收起后每个会话只显示两个**正常并排**的字，不再裁切、不再竖排
  - Ask 页收起后主聊天区**真实变宽**，右侧动作区不被“带过来”
  - 同一套会话显示逻辑覆盖到碳核算页和报告生成页的会话选择区域

## Key Changes
- 抽出共享会话栏组件：
  - 新增一个前端共享 `SessionRail` 组件，用来统一 Ask、碳核算、报告页的“聊天会话列表”展示。
  - 组件职责固定：
    - 头部只保留折叠按钮和纯加号新建按钮
    - 展开态卡片只显示标题
    - 收起态只显示标题前两字缩略
    - 点击切换会话、点击加号新建会话
  - 不再在三个页面里各自维护一套 `List + className` 逻辑。

- Ask 页收起态回退修正：
  - 收起态不再复用当前完整 `Card` 视觉壳层；改成真正的窄 rail，去掉多余 padding、边框、阴影占位。
  - 收起列宽固定到一个足够放下“两字横排缩略”的值，默认采用更宽的 mini rail，而不是现在这种过窄列。
  - 收起态缩略块改成横向短胶囊：
    - 固定最小宽度
    - `white-space: nowrap`
    - 禁止裁切成单字上下堆叠
    - 全部居中
    - 仍保留 tooltip 显示完整标题
  - 展开/收起时主区布局只改左列宽度，右侧内容区继续保持 `minmax(0, 1fr)`，让聊天主区真实扩大；不再保留收起后仍吃宽度的侧栏卡片内边距和壳层空白。

- 会话卡片逻辑统一到多页：
  - Ask、碳核算、报告页的“会话选择区域”全部切到同一套标题优先逻辑。
  - 展开态统一为：
    - 只显示标题
    - 不显示“更新于… / …条消息 / …个附件”等元信息
  - 报告页只替换“会话选择”这一块；“当前会话报告列表”不并入收起缩略逻辑，避免把报告列表误当会话栏处理。
  - 碳核算页和报告页也补同样的折叠按钮与收起态，行为与 Ask 页一致。

- 共享样式收口：
  - 会话栏相关样式改成共享 CSS：
    - 展开 rail
    - 收起 rail
    - 标题卡片
    - 两字缩略胶囊
    - 激活态高亮
  - Ask 页原先只为自己写的 `chat-session-list--collapsed` 规则不再单独主导，避免同类页面继续分叉。
  - 本轮不动主题系统，只保证 V1.1.8G 的浅色/暗色切换下，这套 rail 样式都正常可读。

## Public APIs / Interfaces
- 不新增 API
- 不改：
  - `POST /api/v1/sessions/{id}/ask`
  - `POST /api/v1/sessions/{id}/ask/stream`
- 前端行为契约更新为：
  - Ask、碳核算、报告页的会话选择区共用同一套会话栏组件
  - 展开态会话卡片只显示标题
  - 收起态会话缩略固定为标题前两字横排显示
  - 报告页“报告列表”不属于本轮统一的聊天会话栏范围

## Test Plan
- Ask 页：
  - 收起后每个会话缩略为两个正常并排的字，不裁切、不竖排
  - 收起后主聊天区可见宽度明显变大
  - 右上动作区、消息区、依据抽屉位置不被异常挤动
  - 展开/收起、切换会话、新建会话正常
- 碳核算页：
  - 会话栏改为同样的标题优先逻辑
  - 支持展开/收起
  - 收起态两字缩略显示正常
  - 切换会话、新建会话不退化
- 报告页：
  - “会话选择”区域改为同样的标题优先逻辑
  - 支持展开/收起
  - 收起态两字缩略显示正常
  - “当前会话报告列表”继续正常工作，不误用会话缩略逻辑
- 回归：
  - `npm.cmd run typecheck`
  - `npm.cmd run build`
  - Ask 页现有 streaming / thinking / 滚动归属不退化
  - 暗色模式下新 rail 样式不出现明显断层

## Assumptions
- 本轮继续在当前实验分支推进，不合并到 `main`。
- 统一的是“聊天会话栏”逻辑，不包括报告页里的报告列表。
- 收起态继续只取标题前两字，不引入拼音、首字母或图标替代。
- 默认采用共享组件方案，而不是在三个页面里继续复制粘贴会话栏逻辑。

# CarbonRag V1.1.3 Memory Foundation

## Summary
- 本轮只做 memory foundation，不再扩新业务。
- 分支纪律固定为：
  1. `feature/*` 只用于开发与本地验证
  2. `main` 是稳定源码 baseline
  3. `release/cloud-stable` 是实际部署线，Netlify / VPS 都从这里出
- 本轮核心目标是把 session 做成可压缩、可解释、可预留长期记忆的容器：
  - 单会话自动摘要压缩
  - 最近消息窗口 + session summary 的组合上下文
  - `memory_notes` 只做 backend-only 长期记忆预留
  - 明确 `summary` 和 `knowledge library` 的边界

## Key Changes
- 主线与发布纪律：
  - `main` 作为已经验收通过的稳定源码 baseline
  - `release/cloud-stable` 作为唯一部署线
  - `feature/*` 作为开发分支，不直接当生产线
  - `README.md`、`docs/GIT_RELEASE_FLOW.md`、`docs/deploy/LOCAL_DEV_VS_CLOUD_STABLE.md` 三处同步写死这条纪律

- Session compaction：
  - `session_summary` 作为旧对话的压缩结果
  - `summary_message_seq_upto` 记录摘要覆盖到哪一段消息
  - `summary_updated_at` 记录最近摘要更新时间
  - `summary_estimated_tokens` 记录估算占用，不做精确 tokenizer
  - 只压缩 `user / assistant` 消息，不压缩 `system` 消息
  - 最近窗口保留最近 6 轮内的完整对话
  - ask 运行前先检查 `summary + recent_messages + current question` 的估算占用
  - 压缩失败不阻断问答，只降级为最近窗口或已有 summary

- Memory / knowledge boundary：
  - `knowledge library` 仍然是用户上传和共享样例的知识条目层
  - `memory` 只承载会话过程、上下文压缩结果和用户级长期记忆预留
  - 不把检索片段、知识 chunks 或回答正文回写成知识库存档
  - `memory_notes` 只做后端管理的用户级长记忆预留，不做前端 memory UI

- Ask runtime：
  - `SessionService.build_session_context()` 返回结构化 memory bundle
  - `context_builder` 读取：
    - 最近消息窗口
    - session summary
    - grounding 结果
    - 当前问题
  - `memory_notes` 只作为受控读路径进入 ask 上下文
  - session 页面展示上下文占用估算和是否已压缩，但只标注“估算”

- Backend-only memory_notes：
  - 新增 `memory_notes` 表，归属用户
  - 支持 CRUD，但只在后端接口层保留，不在前端主导航暴露
  - 默认不自动从聊天抽取，不自动写入
  - 只作为后续长期记忆预留位

## Public APIs / Interfaces
- `GET /api/v1/sessions/{id}` 新增 `memory_state`：
  - `context_usage_estimate`
  - `context_budget_estimate`
  - `summary_present`
  - `summary_updated_at`
  - `compacted_message_count`
  - `compaction_status`
- `POST /api/v1/sessions/{id}/ask` 在必要时自动触发 compaction，但请求体不变
- 新增长期记忆预留接口：
  - `GET /api/v1/memory-notes`
  - `POST /api/v1/memory-notes`
  - `PATCH /api/v1/memory-notes/{memory_note_id}`
  - `DELETE /api/v1/memory-notes/{memory_note_id}`
- 分支契约：
  - `main` = 稳定源码 baseline
  - `release/cloud-stable` = 部署线
  - `feature/*` = 开发分支

## Test Plan
- session compaction 触发测试
- compaction 回退测试
- `memory_notes` CRUD 与用户隔离测试
- ask context builder 测试：`recent_messages + session_summary + grounding`
- SQLite / PostgreSQL 双模式下的 summary 与 memory_notes 持久化测试
- 前端 smoke：
  - 连续多轮 ask 后能看到上下文占用估算
  - 超阈值后自动压缩且可继续对话
  - 切换 session 时 memory_state 正确变化

## Assumptions
- 上下文占用显示采用估算值，不承诺精确 token 计数
- 本轮不做长期记忆自动抽取、不做团队级 memory、不做 memory UI
- 本轮不把 memory 和 knowledge library 混成一个概念

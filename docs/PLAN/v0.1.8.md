# CarbonRag v0.1.8 Enterprise Private Sample 接入计划

## Summary
- 基线从当前本地 `feature/conversation-session-foundation` 继续开工，按 `feature/enterprise-private-sample` 落地；起点应包含 `d814530` 这笔 provider 兼容修复。
- v0.1.8 只扩 ask 链路，不碰 `calc-carbon`、`generate-report`、长期 memory、插件系统和完整文件解析。
- 已锁定的实现口径：
  - ask 继续只保留 `POST /api/v1/sessions/{id}/ask`，不恢复顶层 `/api/v1/ask`
  - `mixed` 默认用“均衡配额”检索：`top_k` 尽量按 public/private 平分，奇数余量给 `public_policy`，某一侧不足时再由另一侧补足
  - 文件上传采用策略 A：上传文件只绑定 session 和展示，不进入 retrieval；private retrieval 只吃仓库内 `data/private_sample/` 样例
  - private sample 采用“先选样例再检索”：新增样例 catalog 与 session 关联能力，只有当前 session 已关联的 private sample 参与 `private_sample` / `mixed` 检索
  - 你已确认仓库中文在编辑器里正常，因此 **不做仓库级编码清理**；实现时只修改 v0.1.8 真实触达的内容，不把 PowerShell 乱码当作源文件损坏来大规模修复

## Key Changes
- 数据与检索基线：
  - 建立 `data/private_sample/corpus/{docs,tables,manifest.json}`，至少落 3 份脱敏文档样例和 3 份脱敏 CSV 样例；`README.md` 明确“仅演示样例，不放真实企业原始数据”。
  - `manifest.json` 固定包含：`doc_id`、`title`、`source_type=private_sample`、`sample_type`、`business_topic`、`filepath`、`session_attachable`。
  - 新增 private retrieval 侧实现：loader、chunker、retriever、mixed_retriever；Markdown 继续走段落切分，CSV 采用“每行转结构化自然语言片段”的轻量方案。
  - public/private 两侧最终都要归一到统一 evidence 形状，至少带：`doc_id`、`title`、`source_type`、`source`、`source_url|None`、`snippet`、`chunk_id`、`score`，方便 mixed 合并和 citation 生成。

- 后端 ask / runtime / session：
  - `ask` 继续走 session-scoped 路由；请求体升级为：
    - `question`
    - `knowledge_scope: public | private_sample | mixed`
    - `top_k`
    - `attached_file_ids?: string[]`
  - `attached_file_ids` 仅作为 **当前 session 已关联 private sample 的二次过滤条件**；若为空，则默认使用该 session 当前全部已关联 private sample；上传文件的 `file_id` 本轮不参与 retrieval。
  - ask mode 改为 scope-aware：
    - `public` -> `policy_retrieve`
    - `private_sample` -> `enterprise_retrieve`
    - `mixed` -> `mixed_retrieve`
  - orchestrator 不再固定只跑 `policy_retrieve`；而是根据 `knowledge_scope` 解析工具序列，再统一走 `context_builder -> provider -> response_formatter`。
  - `context_builder` 明确区分三种上下文：
    - `public`：只注入政策片段
    - `private_sample`：只注入脱敏企业样例片段，并明确“不是正式企业审计结果”
    - `mixed`：同时注入 public/private 片段，并在系统提示中要求模型分层回答“政策要求”与“样例现状”
  - session 侧新增 attachment 语义，不再只有上传文件：
    - `SessionDetail` 增加 `attached_files[]`
    - `knowledge_scope_last_used`
    - `source_summary`（可选）
  - `attached_files[]` 统一描述当前 session 的已挂接资产，最小字段固定为：
    - `file_id`
    - `filename`
    - `source_type: uploaded_file | private_sample`
    - `attached_at`
  - 上传文件仍保留现有 `POST /api/v1/files`；本轮只把它映射为 session 的 `uploaded_file` 附件，不进入 retrieval。

- private sample 关联能力：
  - 新增 private sample catalog 接口，供前端列出可挂接样例。
  - 新增 session 级样例关联接口，采用“**整组替换**”语义而不是只追加：
    - 前端每次提交当前选择集
    - 后端直接替换该 session 当前关联的 private sample 集合
    - 这样不需要 v0.1.8 再补 detach 单独接口
  - 当前 session 的 `private_sample` / `mixed` 检索只在该关联集合里查，不扫整个 `data/private_sample/` 仓库。

- citations 与返回结构：
  - `AskCitation` 升级，新增 `source_type: public_policy | private_sample`
  - `source_url` 对 private sample 允许为 `null`
  - `AskResponse` 增加顶层 `source_summary`，最小字段固定为：
    - `knowledge_scope`
    - `public_policy_count`
    - `private_sample_count`
    - `total_citation_count`
  - mixed 返回必须能同时包含两类 citation，并且 counts 与 citations 保持一致。

- 前端 workbench：
  - 保留当前 conversation workbench，不回退到单页表单。
  - 输入区新增 scope 控件，默认 `public`。
  - 新增 private sample catalog / attach UI，推荐做成输入区附近按钮 + drawer 或 panel，允许当前 session 选择样例。
  - 当前 session 状态区要明确展示：
    - 当前 scope
    - 已上传附件数
    - 已关联 private sample 数
  - citations 面板升级：
    - public/private 标签颜色区分
    - mixed 时按来源分组展示
    - snippet 支持折叠
    - 回答区或面板顶部展示 `source_summary`
  - 当前上传文件继续显示，但文案要明确“本轮上传文件不会参与 ask，仅作为 session 绑定资产”。

- 文档与研究同步：
  - `README.md` 推进到 `v0.1.8 enterprise-private-sample 接入中`
  - `docs/API_BOUNDARY_DRAFT.md` 同步 ask 三种 scope、session 附件与 new citation/source_summary
  - 新增：
    - `docs/PLAN/v0.1.8.md`

## Public APIs / Interfaces
- 保留并升级：
  - `POST /api/v1/sessions/{id}/ask`
    - 请求体：`question`、`knowledge_scope`、`top_k`、`attached_file_ids?`
    - 响应体：现有 `answer / mode / status / citations / trace_id` 保持，新增 `source_summary`
- 新增：
  - `GET /api/v1/private-samples`
    - 返回 attachable private sample catalog
  - `PUT /api/v1/sessions/{id}/attached-files/private-samples`
    - 请求体：`{ "doc_ids": ["enterprise_doc_001", "production_sample_001", ...] }`
    - 语义：替换该 session 当前关联的 private sample 集合
- `SessionDetail` 升级：
  - 新增 `attached_files[]`
  - 新增 `knowledge_scope_last_used`
  - 新增 `source_summary?`
  - 现有 `files[]` 可在实现时保留为上传资产兼容字段，但前端与文档以 `attached_files[]` 为主口径
- `AskCitation` 升级：
  - `source_type: public_policy | private_sample`
  - `source_url: string | null`

## Assumptions
- v0.1.8 从当前 `feature/conversation-session-foundation` 继续开新分支，而不是回退到 v0.1.6 或更早分支。
- 上传文件沿用策略 A：本轮不解析用户上传文件，不把上传文件转成 private transient sample。
- mixed 默认采用“均衡配额 + 奇数余量给 public + 一侧不足由另一侧补足”。
- 当前 PowerShell/终端读中文可能出现乱码，但这不被视为 repo 全局编码问题；本轮不单独做编码清理任务。

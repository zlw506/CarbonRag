# CarbonRag v0.1.6 public-policy 首个受控检索接入轮计划

## Summary
- 从 `feature/ask-mode-controlled-link` 切出 `feature/public-policy-grounded-ask`，先提交 `checkpoint: before v0.1.6`，最终提交 `v0.1.6: ground ask mode with public policy retrieval and citations`。
- 本轮只升级 ask 链路：前端问答页 -> `POST /api/v1/ask` -> ask mode -> 固定 `policy_retrieve` -> 注入公共政策片段 -> provider 回答 -> 返回真实 `citations`。
- 已锁定三项关键实现口径：
  - 检索底座采用 `jieba + BM25`，不做向量库。
  - `knowledge_scope` 在 v0.1.6 只允许 `public`；`mixed` 和 `private_sample` 都返回 `422`。
  - 检索为空时仍返回结构化受限回答，HTTP `200`、`status="ok"`，但必须明确说明未检索到足够政策依据，`citations` 可为空。

## Key Changes
- 公共政策语料基线：
  - 在 `data/public/corpus/` 新增 `docs/` 和 `manifest.json`，至少落 5 份真实官方公开政策样本，每份使用 Markdown frontmatter 固定 `doc_id/title/source/source_url/issued_at/region/doc_type`。
  - 语料集固定为 5 份人工整理的官方摘录，不放 PDF 原文，不做自动抓取：
    - 《国务院关于印发2030年前碳达峰行动方案的通知》，国务院，`region=national`，`doc_type=policy`
    - 《关于加快建立统一规范的碳排放统计核算体系实施方案》，国家发展改革委/国家统计局/生态环境部，`region=national`，`doc_type=guideline`
    - 《建立健全碳达峰碳中和标准计量体系实施方案》，市场监管总局等九部门，`region=national`，`doc_type=guideline`
    - 《关于进一步强化碳达峰碳中和标准计量体系建设行动方案（2024—2025年）的通知》，相关国家部门，`region=national`，`doc_type=guideline`
    - 《北京市人民政府关于印发〈北京市碳达峰实施方案〉的通知》，北京市人民政府，`region=beijing`，`doc_type=policy`
  - 每份样本正文用自然段摘录 4 到 8 段可问答片段，内容围绕“双碳目标、碳达峰、碳中和、标准计量、统计核算、北京地方实施”等高频问答主题。
- 检索与工具层：
  - 新增 `backend/app/retrieval/`，拆为 `public_corpus_loader.py`、`public_chunker.py`、`public_retriever.py`、`schemas.py`。
  - `public_corpus_loader` 负责读取 `manifest.json`、校验元信息、加载 markdown 正文并返回统一文档对象。
  - `public_chunker` 固定采用“按空行分段 + 合并过短段落”的稳定切分：去空段，按顺序合并到至少约 80 字，单 chunk 上限约 420 字，生成稳定 `chunk_id`，格式固定为 `policy_001_chunk_01`。
  - `public_retriever` 采用 `jieba.lcut_for_search` 分词 + BM25，对每个 chunk 建索引；查询时对 question 做相同分词，仅返回正分命中，按分数降序取 `top_k`；标题命中可加轻微 boost，`region/doc_type` 过滤先保留接口但本轮默认不开放前端入口。
  - `policy_retrieve_stub.py` 升级为真实工具 `policy_retrieve.py`，并在 registry 中替换注册；输出固定为检索结果对象和 citation 元信息，不再返回 stub 文本。
- ask mode / orchestrator / provider：
  - ask mode 固定改为单工具受控模式：`allowed_tools=("policy_retrieve",)`，`default_stub_tool_sequence=("policy_retrieve",)`，不得自动规划其他工具。
  - `orchestrator.run()` 对 ask mode 的流程固定为：校验请求 -> 调 `policy_retrieve` -> 拿到 retrieval results -> 交给 `context_builder` -> 调 provider -> `response_formatter` 产出答案与 citations。
  - `context_builder` 为 ask mode 注入五类信息：系统角色、模式说明、回答限制、检索到的政策片段列表、用户问题。若检索为空，system prompt 必须显式写明“未检索到足够政策依据，不得伪造引用，只能给出受限说明”。
  - provider 层保持现有 OpenAI-compatible 结构，不新增第二套 provider；只把检索上下文塞进现有 chat provider 的 `system` prompt。
  - `response_formatter` 不再让 `citations` 由路由层硬编码为空，而是从 retriever 的真实结果直接映射；模型输出只负责 answer，不负责 citation 生成。
- ask 路由与前端：
  - `POST /api/v1/ask` 继续沿用原路由，不新增第二条 ask 接口；路由层将 `knowledge_scope` 严格限制为 `public`，非 `public` 返回 `422 invalid_input`。
  - `AskResponse.citations` 的公开结构升级为：
    - `doc_id`
    - `title`
    - `source`
    - `source_url`
    - `snippet`
    - `chunk_id`
  - 前端继续只改问答页；在现有答案展示基础上新增 citation 列表，逐条显示标题、发布机构、片段摘要、来源链接或来源标识。
  - 页面文案同步推进为“已接入公共政策样本检索；citations 来自本地公共政策样本语料；仍未接企业私有数据与完整知识库”。

## Public Interfaces / Types
- `POST /api/v1/ask` 请求体保持：
  - `question: string`
  - `knowledge_scope: "public" | "private_sample" | "mixed"`
  - `top_k: number`
- `POST /api/v1/ask` 的运行时行为改为仅接受 `knowledge_scope="public"`：
  - `public` 正常处理
  - `mixed` 返回 `422`
  - `private_sample` 返回 `422`
- `POST /api/v1/ask` 响应体保持顶层 shape 不变，但 `citations` 字段从空占位升级为真实结构：
  - `answer: string`
  - `mode: "ask"`
  - `status: "ok" | "invalid_input" | "provider_error"`
  - `citations: Array<{ doc_id, title, source, source_url, snippet, chunk_id }>`
  - `trace_id: string`
- HTTP 语义保持：
  - 输入非法：`422`
  - provider 失败：`502`
  - 成功：`200`
- 内部新增 retrieval 契约：
  - `RetrievedChunk` 至少包含 `doc_id/title/source/source_url/issued_at/region/doc_type/chunk_id/snippet/score`
  - `policy_retrieve` 的输出直接复用 retrieval schema，不允许模型自行生成 citation 字段

## Assumptions
- 本轮允许为后端新增轻量依赖 `jieba` 与 `rank-bm25`，不引入 embedding、向量库或数据库服务。
- 语料采用“真实官方来源 + 人工整理摘录”策略，source_url 必须是官方公开链接，正文可以是结构化摘录而非全文。
- 检索为空不视为 provider 错误；接口仍返回 `200` 和 `status="ok"`，但答案必须受限且 citations 允许为空。
- v0.1.6 不改 `calc-carbon`、`generate-report`、企业私有数据、memory、插件系统和自动爬虫；只升级 public-policy grounding 这一条 ask 链路。

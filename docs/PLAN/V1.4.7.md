# V1.4.7 GitNexus 全量试用与 Codex-MCP 结构感知工作流冻结计划

## Summary
- 本轮不做业务功能、不改 UI、不碰 RAG/碳核算主线；只把 GitNexus 接入 #1 本地开发与 PR 审查流程。
- 当前主线已确认在 `main`，本轮从 `t1/v1.4/gitnexus-adoption-pilot` 执行。
- 上游核验结果：npm `gitnexus` latest 为 `1.6.3`，但本机实测会崩溃；社区 issue #1406 维护者建议使用 rc，故本轮冻结为 `gitnexus@1.6.4-rc.84`。仓库指向 `abhigyanpatwari/GitNexus`，许可证为 `PolyForm-Noncommercial-1.0.0`。
- 本轮冻结关系：OpenSpec 管需求边界，GitNexus 管代码结构感知，Codex 读取两者后再执行修改。

## Key Changes
- 从 `main` 新建 `t1/v1.4/gitnexus-adoption-pilot`，创建 OpenSpec change：`gitnexus-code-intelligence-pilot`，影响 `governance`、`devops-release`，必要时补 `ai-runtime/conversation-memory/knowledge-rag/carbon-report-feedback` 说明但不改业务代码。
- 安装与配置流程固定为：全局安装 `gitnexus@rc`，用 `codex mcp add gitnexus -- npx -y gitnexus@rc mcp` 注册 MCP，再用 `codex mcp list` 确认。
- 全量索引必须跑满 6 项：`.gitnexus/lbug`、agent context、BM25、`--embeddings`、`--skills`、`--verbose`，统一脚本为 `scripts/gitnexus-full-index.ps1` / `scripts/gitnexus-full-index.sh`。
- 新增一键脚本 `scripts/gitnexus-full-index.ps1` 与 `scripts/gitnexus-full-index.sh`，只封装安装后索引、状态检查、列表检查，不提交 `.gitnexus/` 或 `logs/gitnexus/`。
- 更新根 `AGENTS.md`，新增“GitNexus code intelligence rule”，但必须保留现有 OpenSpec-first 规则；如果 GitNexus 自动改写 `AGENTS.md/CLAUDE.md`，实施者必须人工审查后再纳入。
- 新增 GitNexus 文档：运行手册、Codex MCP checklist、图谱可视化研究、V1.4.7 计划；更新 OpenSpec/Codex 工作流、PR Review、README、`.gitignore`。
- `.gitignore` 必须新增并验证：`.gitnexus/`、`logs/gitnexus/`，保留现有 `.spec-gen/`、`3rdparty/spec-gen/`、runtime data 忽略规则。

## Verification
- 安装与索引验证：`gitnexus --version`、`codex mcp list`、`scripts/gitnexus-full-index.ps1 -Proxy "http://127.0.0.1:17891"`、`gitnexus status`、`gitnexus list`。
- 完整索引验收：确认 `.gitnexus/lbug` 存在；日志中能看到 embeddings、skills、verbose 诊断；记录生成的 module-level skills 实际路径。
- Web UI 验收：运行 `gitnexus serve`，确认能看到 CarbonRag repo、nodes/edges/clusters/processes，并能定位 `backend/app/carbon`、`frontend/src/pages`、session/report/knowledge 相关节点或关系。
- MCP 使用验收：在 Codex 中通过 GitNexus MCP 至少执行一次 repo 列表/上下文查询；对 `backend/app/carbon` 做一次影响范围分析；对一个模拟 PR diff 做一次 `detect_changes` 或等价影响分析。
- 回归验证：`openspec validate --all` 必须通过；本轮不要求后端/前端全量业务测试，除非 GitNexus 脚本或文档变更触发 CI 失败。

## Assumptions
- GitNexus 只作为本地开发增强与 PR 审查辅助，不作为 CarbonRag 产品运行时依赖。
- GitNexus 的 UI/图谱实现因许可证限制只做研究参考，不复制源码或图渲染代码。
- 若 `--embeddings` 因本机网络、模型或 GitNexus 上游问题失败，本轮不能降级验收；必须记录失败原因并修复后重跑。
- 若 VS Code Codex 插件无法直接显示 MCP 工具调用结果，以 Codex CLI 的 MCP 调用验证为最低可接受证据，并把 VS Code 插件状态写入 checklist。
- 上游参考：GitHub issue #1406 确认 `1.6.3` 的 embeddings 失败建议使用 rc；npm registry 确认 `gitnexus@1.6.4-rc.84`、仓库与许可证。

# CarbonRag V1.2.1 OpenSpec 试点接入 + 多团队 Git 工作流升级

## Summary

本轮不做业务功能、不改 UI、不碰 AI 聊天主链路，只落工程治理：

- OpenSpec 本地试点与首版规格库
- PR / Review / CODEOWNERS
- #1 / #2 多团队分支纪律
- M1-M8 模块边界
- 版本、发布、文档纪律

## Decisions

- `Git-ys1` 是云端 `main` 最终管理员和 PR 最终审查人。
- `main` 是稳定源码和当前公网默认发布基线。
- `release/cloud-stable` 暂时保留为兼容线，不再作为默认发布线。
- #2 初期采用 fork-and-PR。
- OpenSpec/spec-gen 先由 #1 本地试点，人工校验后再让 #2 团队执行。

## Deliverables

- `openspec/config.yaml`

- `openspec/changes/**`
- `.github/PULL_REQUEST_TEMPLATE.md`
- `.github/CODEOWNERS`

## Acceptance

- README 状态推进到 V1.2.1。
- 版本规则写入文档：`VA.B.0` 是总监规划轮，`VA.B.C` 是执行轮。
- PR 模板覆盖 OpenSpec、模块、风险、验证和批准。
- CODEOWNERS 初版覆盖 M1-M8。

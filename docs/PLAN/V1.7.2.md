# V1.7.2 通用文件预览抽屉

## Summary
- 新增统一“文件预览”能力，覆盖聊天附件、知识库文档、RAG citation、爬虫候选文件，不再只看片段。
- 后端只接受逻辑 ID，不暴露真实 `storage_path`；前端统一用一个 Drawer 悬浮窗口打开。
- 本轮优先展示“解析后的 Markdown / 文本 / 片段 / 元数据 / 原始文件下载或内嵌预览”，不做 Office 原版在线排版渲染。

## Key Changes
- 新建 OpenSpec change：`universal-file-preview-drawer`，新增 `docs/PLAN/V1.7.2.md` 和 `docs/architecture/UNIVERSAL_FILE_PREVIEW_ARCHITECTURE.md`。
- 新增后端统一接口：
  - `GET /api/v1/file-previews/{source_type}/{source_id}`
  - `GET /api/v1/file-previews/{source_type}/{source_id}/raw`
- `source_type` 固定支持：
  - `session_file`：聊天页上传文件，`source_id=file_id`
  - `rag_document`：知识库文档，`source_id=doc_id`，必传 `kb_id`
  - `crawler_candidate`：爬虫候选文件，`source_id=candidate_id`，仅管理员
  - `knowledge_item`：旧知识条目兼容入口，`source_id=knowledge_item_id`
- 预览内容统一返回：
  - 文件标题、文件名、MIME、大小、状态、来源 URL、页码/sheet/slide 摘要
  - `markdown`：优先 `file_parse_results.extracted_markdown`、RAG 文档 `metadata.parsed_markdown`、crawler `metadata.markdown_storage_path`
  - `text`：优先解析文本，fallback 到 cleaned/raw 文本
  - `chunks`：聊天附件关联 knowledge chunks、RAG chunks、crawler 发布后的 RAG chunks
  - `metadata`：解析器、hash、source、candidate/run、错误阶段等
  - `raw_preview_url`：PDF、图片、HTML、TXT/MD 可内嵌；DOCX/XLSX/PPTX 默认提供下载和解析预览
- 权限规则：
  - 聊天文件只能 owner 或 admin 查看。
  - RAG 文档按 KB 可见性和 owner 校验。
  - crawler candidate 只允许 admin 查看。
  - 后端禁止前端传路径，所有路径由服务端按 ID 解析并做目录边界校验。

## Frontend
- 新增通用组件 `FilePreviewDrawer` 和 service `filePreview.ts`。
- Drawer 统一标签页：
  - `解析预览`：Markdown 渲染。
  - `纯文本`：可复制全文。
  - `检索片段`：展示 chunk、页码/sheet/slide、vector 状态。
  - `原始文件`：PDF/图片/HTML/TXT 可内嵌，Office 提供下载。
  - `元数据`：source、hash、parse/index 状态、错误信息。
- 接入位置：
  - AskPage：用户消息附件 chip、citation 卡片增加“查看文件”。
  - KnowledgeBaseWorkbench：文档列表、片段预览、citation/test QA 结果增加“查看文件”。
  - Admin 实时爬虫：candidate 列表增加“查看抓取文件”，并保留“打开原网页”。
- 对 crawler 预览额外显示：原网页 URL、canonical URL、抓取时间、raw/cleaned/markdown hash，方便和真实网页对比。

## Test Plan
- 后端测试：
  - `session_file` 预览能返回 markdown/text/chunks，非 owner 404。
  - `rag_document` 预览要求 `kb_id`，跨 KB/跨用户不可读。
  - `crawler_candidate` 普通用户 403，admin 可读 raw/cleaned/markdown。
  - raw endpoint 拒绝路径穿越，只允许已登记文件。
- 前端验证：
  - 聊天上传 DOCX/PDF 后点击附件能打开预览抽屉。
  - 知识库文档入库后点击“查看文件”能看到全文解析和 chunks。
  - 爬虫 candidate 点击“查看抓取文件”能看到 markdown，并可打开原网页对比。
- 回归命令：
  - `openspec validate universal-file-preview-drawer --strict`
  - `openspec validate --all`
  - 后端 targeted pytest
  - `cd frontend && npm.cmd run typecheck && npm.cmd run build`
  - `git diff --check`

## Assumptions
- 本轮不做完整 Office 在线排版渲染；DOCX/XLSX/PPTX 先展示解析 Markdown/text/chunks，并提供原文件下载。
- 爬虫候选的 `markdown_storage_path / cleaned_storage_path / storage_path` 只由服务端读取，前端不接触真实路径。
- GitNexus 当前索引 stale，只可辅助理解；实施前应重建索引再做 impact/detect_changes。

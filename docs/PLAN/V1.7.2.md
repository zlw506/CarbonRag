# V1.7.2 Crawler Source Registry and RAG Quality Loop

## 目标

把现有 policy crawler 从“固定内置源 + 手动候选发布”升级为“推荐官方源库 + 管理员自定义源 + dry-run 预览 + 质量评分 + 发布到 RAG KB + 检索验收”的闭环。

## 正式链路

```text
推荐/自定义官方 source
-> dry-run 预览
-> 手动运行 crawl
-> candidate artifact
-> candidate_quality_score
-> publish-to-rag
-> 官方政策自动更新库
-> quick pipeline
-> /rag/search / AskPage 可引用
```

## 本轮完成口径

- `scheduled=false` 与 `auto_publish=false` 继续作为安全默认值。
- 管理员可创建、更新、删除官方 source，并可导入推荐双碳源。
- `dry-run` 不写库、不入 RAG，只返回候选预览、跳过原因、markdown 预览与质量分。
- candidate metadata 记录 `candidate_quality_score / quality_breakdown / matched_keywords / skip_reason`。
- 质量分低于 60 的 candidate 禁止直接 `publish-to-rag`。
- 管理后台“实时爬虫”升级为“自动更新源”。

## 非目标

- 不引入 Crawlab master-worker / MongoDB / SeaweedFS。
- 不用 Crawl4AI 替换 Scrapy。
- 不打开自动定时抓取和自动发布。

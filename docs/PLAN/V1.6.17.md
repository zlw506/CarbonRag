# V1.6.17 本地 Ollama / DeepSeek Provider Runtime 接入

## Summary
- 目标：让 CarbonRag 在 local-dev 下使用本机 Ollama `deepseek-r1:8b` 生成回答，覆盖 AskPage、`/rag/answer`、KnowledgeBaseWorkbench Test QA。
- Ollama native API 作为主路线：`http://localhost:11434/api/chat`；OpenAI-compatible `http://localhost:11434/v1/chat/completions` 作为兼容路线。
- 云端 Netlify + VPS 默认不能访问用户电脑的 `localhost:11434`，本轮本地开源模型能力只保证本机开发环境。

## Key Changes
- 后端新增/重构 Ollama provider：新增 `ollama_client.py`，补强现有 Ollama chat provider，支持 `model/messages/stream/temperature/num_ctx/keep_alive/think`，并保留 Ollama `message.thinking` 到 trace/stream metadata。
- 修改 provider factory：按 `provider_type` 分派 `ollama / openai_compatible / carbonrag_cloud / deepseek`，不再固定返回 OpenAI-compatible；`ollama` 允许空 `api_key`，默认 `base_url=http://localhost:11434`、`model=deepseek-r1:8b`。
- 打通 RAG 生成 provider：`/rag/test-qa` 与 `/rag/answer` 必须使用当前用户 active provider；AskPage 已有 active provider 链路，补测试锁死 `kb_id/rag_mode/provider` 都进入 runtime。
- 前端设置页增加模板按钮：`一键填充 Ollama / DeepSeek-R1 8B`，自动填 `provider_type=ollama`、`base_url=http://localhost:11434`、`model_name=deepseek-r1:8b`、`api_key=` 空，并复用模型刷新/测试 provider 能力。
- 新增运行资产：`.env.llm.ollama-local.example`、`scripts/llm-ollama-smoke.ps1/.sh`、`docs/architecture/LOCAL_LLM_PROVIDER_ARCHITECTURE.md`、`docs/PLAN/V1.6.17.md`，并更新公告/README，写清 local-dev 与云端边界。

## Public Interfaces / Config
- 新增环境配置默认：
  - `AI_CHAT_PROVIDER=ollama`
  - `OLLAMA_BASE_URL=http://localhost:11434`
  - `OLLAMA_MODEL=deepseek-r1:8b`
  - `OLLAMA_TIMEOUT_SECONDS=180`
  - `OLLAMA_NUM_CTX=8192`
  - `OLLAMA_KEEP_ALIVE=10m`
  - `OLLAMA_THINK=true`
  - `RAG_GENERATION_PROVIDER=ollama`
  - `RAG_GENERATION_MODEL=deepseek-r1:8b`
- `/rag/test-qa`、`/rag/answer`、Ask metadata 透出 `provider=ollama`、`model=deepseek-r1:8b`、`provider_ref`、`thinking` 摘要或 trace 字段。
- 保留一键切回云端：active provider 改回 `carbonrag_cloud` 或 `openai_compatible` 即可，不删除既有云端配置。

## Test Plan
- 后端测试：Ollama native payload、streaming thinking、OpenAI-compatible fallback、provider factory 分派、active provider 被 Ask/RAG Test QA/RAG Answer 使用。
- Smoke：`scripts/llm-ollama-smoke.ps1` 调 `/api/tags` 和 `/api/chat`，确认 `deepseek-r1:8b` 能回答 `OK`。
- RAG 验收：用青木制造知识库问“2025 年第一季度合计外购电力是多少”，必须回答 `217,650 kWh`，citation 非空，trace 显示 `provider=ollama`、`model=deepseek-r1:8b`、`vector_runtime=milvus_standalone`、`degraded=false`。
- 反事实验收：问“2025 年第二季度外购蒸汽是多少”，必须说明知识库无依据，不能编造。
- 回归命令：`openspec validate local-ollama-llm-provider-runtime --strict`、`openspec validate --all`、后端 targeted pytest、前端 `typecheck/build`。

## Assumptions
- 用户本机已安装 Ollama，且本地模型 tag 精确为 `deepseek-r1:8b`；若 tag 不同，只允许文档说明别名，不改变本轮验收模型名。
- 本轮不改 BGE-M3 embedding、Milvus Docker、reranker，不扩 RAG-Pro 字段，不做碳核算。
- 参考 Ollama 官方 API：默认 base URL 为 `http://localhost:11434/api`，chat endpoint 为 `/api/chat`，响应包含 `message.content` 和可选 `message.thinking`；OpenAI 兼容入口使用 `http://localhost:11434/v1/`。

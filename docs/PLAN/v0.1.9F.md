# CarbonRag v0.1.9F 双环境固化与发布纪律轮

## 目标
- 固化 `local-dev` 与 `cloud-stable` 两种运行模式
- 固化 `release/cloud-stable` 发布分支
- 固化本地启动脚本与环境文件策略
- 固化 Netlify / VPS 的稳定发布口径

## 本轮不做
- 不加新业务功能
- 不改 Netlify / VPS 架构
- 不改 ask / calc / feedback 接口契约
- 不处理长期 memory、report、插件系统

## 关键交付物
- `frontend/.env.production`
- 本地开发模板化的 `frontend/.env.example`
- `scripts/dev-local.ps1`
- `scripts/dev-local.sh`
- `docs/deploy/LOCAL_DEV_VS_CLOUD_STABLE.md`
- `docs/GIT_RELEASE_FLOW.md`
- 更新后的 README、GIT_WORKFLOW、DEVELOPMENT_BOOTSTRAP、Netlify/VPS 部署文档

## 验收口径
- 本地 dev 默认命中 `127.0.0.1:8000/api`
- 本地后端强制 SQLite fallback
- 云端稳定发布默认走 `release/cloud-stable`
- Netlify 与 VPS 文档口径一致
- 前端构建不包含本地 API 地址

# V1.4.5 Carbon Inventory Trust Baseline 计划

## Summary
- 本轮目标：把 V1.4.4 的“因子驱动演示计算器”升级为“可回放、可审计、可解释、能接官方因子”的企业基础碳核算底座。
- 本轮不扩前端 UI、不做复杂行业公式、不做完整 Scope 3、不做自动同步国家因子库；只做后端、数据、OpenSpec、文档和测试。
- 权威来源锚定：MEE/NBS 2023 电力因子公告、JS/T 303-2026 公共机构核算指南、GB/T 32150-2025、GHG Protocol Scope 2 Guidance、EPA Scope 1/2 inventory guidance。
- OpenSpec change 固定为：`carbon-inventory-trust-baseline`，影响 `carbon-report-feedback` spec。

## Key Changes
- 新增可信持久化模型：保留 `carbon_calculations` 兼容旧查询，同时新增 `carbon_inventories`、`carbon_activity_items`、`carbon_calculation_lines`、`carbon_factor_snapshots`、`carbon_evidence_references`、`carbon_inventory_summaries`；SQLite/PostgreSQL schema 同步补齐。
- `POST /api/v1/calc-carbon` 保持兼容，但内部主事实源改为 `CarbonInventory`；旧三字段会转换为非零 activity item，`0` 值不再生成天然气/柴油等 activity，也不再产生无关 demo warning。
- `CarbonActivityItem` 扩展证据字段：`province`、`data_quality`、`evidence_reference`、`source_document_id`、`entry_method`、`requested_factor_id`；默认 `data_quality=user_input`、`entry_method=manual`。
- 因子数据拆分新增：
  - `data/factors/electricity_cn_2023_official.json`：录入全国 0.5306、剩余组合 0.6096、化石能源 0.8273、七大区域、31 个省级因子。
  - `data/factors/fuel_combustion_cn_guidance_seed.json`：录入 JS/T 303-2026 常用燃料缺省值，`source_type=guidance_default`，不得伪装 official。
  - `carbon_v2_seed.json` 继续保留兼容，但加载器改为加载多文件 factor sources。
- `FactorRecord` / `FactorRegistry` 升级字段：`source_priority`、`effective_year`、`region_level`、`method_type`、`applicable_industry`、`applicable_standard`、`quality_level`、`is_official`、`valid_from`、`valid_to`。
- 因子选择优先级写死：`requested_factor_id` > 同地区同年份官方 > 同地区最近年份官方 > 上级地区官方 > `guidance_default` > `demo`；任何 fallback 必须进入 `warnings[]`。
- 响应新增并稳定返回：`inventory_id`、`total_kgco2e`、`scope_summary`、`activity_count`、`official_factor_count`、`fallback_factor_count`、`warnings`；继续返回现有 `trace_id / total_emission_kgco2e / breakdown / citations / factor_snapshot / unit_conversion_trace / formula_trace / source_summary`。
- Scope 2 market-based 本轮仍不完整计算，但保留 `scope2_market_kgco2e=null`；如用户传绿电字段，只做 evidence/warning 记录，不把绿电随意算零。

## Public APIs / Types
- `POST /api/v1/calc-carbon` 请求兼容旧三字段，并扩展 `activity_items[]`：
  - activity item 新增：`province?`、`data_quality?`、`evidence_reference?`、`source_document_id?`、`entry_method?`、`requested_factor_id?`。
  - `data_quality` 允许：`metered | invoice | estimate | user_input | demo`。
  - `entry_method` 允许：`manual | file_upload | imported | api | demo`。
- 响应新增：
  - `inventory_id: string`
  - `total_kgco2e: number`
  - `scope_summary: { scope1_kgco2e, scope2_location_kgco2e, scope2_market_kgco2e, scope3_reserved_kgco2e }`
  - `activity_count: number`
  - `official_factor_count: number`
  - `fallback_factor_count: number`
- `GET /api/v1/sessions/{id}/carbon-calculations` 仍返回现有摘要，不破坏 report 选择 calc result；内部可从新 inventory 表补充读取。

## Test Plan
- 新增/更新后端测试：
  - `test_carbon_inventory_persistence.py`：V2 原始请求、activity raw payload、inventory metadata 可回放。
  - `test_carbon_legacy_zero_filter.py`：只填 electricity 不生成 gas/diesel activity，不产生无关燃料 warnings。
  - `test_carbon_electricity_cn_2023_factors.py`：national/grid_region/province/method_type 选择正确。
  - `test_carbon_fuel_guidance_seed.py`：燃料 guidance_default 可计算且带适用范围 warning。
  - `test_carbon_factor_priority.py`：验证 requested > official exact > official fallback > guidance_default > demo。
  - `test_carbon_inventory_summary.py`：scope1、scope2_location、market reserved、official/fallback counts 正确。
  - `test_carbon_evidence_fields.py`：data_quality、entry_method、evidence_reference、source_document_id 持久化。
- 回归测试：
  - 旧三字段接口仍能算并绑定 session/report。
  - `factor_snapshot` 保存计算当时完整因子值，不只保存 factor_id。
  - SQLite/PostgreSQL bootstrap 都创建新表和迁移列。
  - `openspec validate carbon-inventory-trust-baseline --strict`、`openspec validate --all`、`cd backend && python -m pytest` 必须通过。

## Assumptions
- 采用完整明细表方案，不走单表 JSON 扩展。
- 本轮完整人工录入 2023 电力因子与燃料 guidance seed；若省级因子录入时发现官方 PDF 表格解析存在歧义，必须保留原文来源行并在测试中锁定已校验值。
- 国家温室气体排放因子数据库本轮只设计 `manual_official_factor_import` 结构，不做自动爬取/API 同步。
- 本轮严格不改前端 UI；如前端已有字段展示不了新增响应，也不在本轮处理。
- `GB/T 32150-2025` 作为企业组织层边界依据进入文档与 spec，不在本轮实现完整标准校验器。

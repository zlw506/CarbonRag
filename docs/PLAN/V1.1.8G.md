# CarbonRag V1.1.8G 会话缩略修正 + 全站主题配置入口

## Summary
- 基线继续使用当前实验分支 `feature/v1.1.8b-ux-audit-remediation`，不合并到 `main`。
- 本轮只做两类前端热修：
  - 修正 Ask 页左侧收起态会话缩略的排版，让每个会话保持“两字同排”，不再竖排挤压
  - 增加全站主题配置入口，提供 `浅色 / 暗色 / 跟随系统` 三档，并持久化到浏览器本地
- 当前已知现状：
  - 收起态会话缩略来自 `session.title.slice(0, 2)`，但容器过窄过高，导致两个汉字被竖向挤开
  - 用户入口已经被改到左侧 `Sider`，可直接作为主题配置入口承载位
  - 仓库当前没有 `ConfigProvider` 主题算法、没有主题状态、没有本地主题持久化

## Key Changes
- Ask 页收起态会话缩略修正：
  - 保持当前“只显示前两个字”的策略，不改文案来源
  - 把收起态缩略从“窄高竖胶囊”改成“短宽横胶囊/圆角块”，确保两字始终同排
  - `chat-session-list__mini` 改为固定横向布局：
    - 足够的最小宽度容纳两个汉字并排
    - 居中对齐，不允许换行到竖排
    - 溢出时仍只取前两字，不再靠容器挤压
  - Ask 左侧会话栏收起态的按钮区一并收口：
    - 折叠按钮和新建按钮继续保留
    - “新建会话”维持纯加号
    - 不恢复任何辅助说明或元信息

- 全站用户入口与主题配置统一：
  - 当前左侧 `Sider` 常驻头像入口保留，并扩展为所有页面统一的“用户与主题设置入口”
  - 不恢复右上角用户菜单；Ask 与其他页面都继续走左侧入口
  - 用户菜单内容固定为：
    - 用户名
    - 角色标签
    - 主题设置：`浅色 / 暗色 / 跟随系统`
    - 管理后台入口（仅 admin）
    - 退出登录
  - 主题切换默认即时生效，不需要刷新页面

- 主题系统最小落地方式：
  - 在 `frontend/src/app/App.tsx` 之上或内部补一个轻量主题状态层，例如 `ThemeProvider`
  - 用浏览器 `localStorage` 持久化主题偏好，键名固定为仓库内统一前缀，例如 `carbonrag-theme-mode`
  - 三档规则固定为：
    - `light`：强制浅色
    - `dark`：强制暗色
    - `system`：跟随 `prefers-color-scheme`
  - Ant Design 主题通过 `ConfigProvider` 接入：
    - 浅色用 `defaultAlgorithm`
    - 暗色用 `darkAlgorithm`
  - 自定义样式继续以全局 CSS 为主，但改为基于 `data-theme` 或等效根节点标记切换变量，避免只切 antd 不切自定义界面
  - 第一轮只覆盖当前核心页面壳层和高频组件：
    - `AppShell`
    - Ask 页
    - 会话栏
    - composer
    - 卡片/抽屉/消息气泡
  - 不要求本轮把每个页面都做成完整视觉重设计，但至少不能出现明显的亮暗混搭断层

## Public APIs / Interfaces
- 不新增后端 API
- 不改：
  - `POST /api/v1/sessions/{id}/ask`
  - `POST /api/v1/sessions/{id}/ask/stream`
- 新增前端本地配置契约：
  - 主题模式：`light | dark | system`
  - 本地持久化：浏览器 `localStorage`
- 用户入口行为契约：
  - 全站统一从左侧头像入口进入
  - 主题配置与用户菜单共用同一入口

## Test Plan
- Ask 页收起态回归：
  - 左侧会话栏收起后，每个缩略卡片的两个汉字保持同排
  - 不再出现竖排、挤压、单字上下堆叠
  - 收起态切换会话、新建会话、展开会话栏仍正常
- 主题配置回归：
  - 左侧头像入口在 Ask、知识库、碳核算、报告页都可见且可点
  - `浅色 / 暗色 / 跟随系统` 三档切换都即时生效
  - 刷新页面后主题偏好仍保留
  - `system` 下切换系统主题时，页面能同步切换
  - admin 仍可见后台入口，普通用户不可见
- 视觉回归：
  - 暗色模式下壳层、侧栏、卡片、composer、消息气泡、抽屉不出现明显亮暗混搭
  - Ask 页现有流式、thinking、滚动归属、上下文小圆按钮不退化
- 构建检查：
  - `npm.cmd run typecheck`
  - `npm.cmd run build`

## Assumptions
- 本轮继续在当前实验分支推进，不合并到 `main`
- 主题配置入口只放在左侧头像菜单内，不额外新增独立设置页
- 收起态会话缩略继续只取标题前两字，不额外引入拼音、首字母或图标编码
- 暗色模式本轮优先做“配置可用 + 核心界面不割裂”，不做完整品牌视觉重构

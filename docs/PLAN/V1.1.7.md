# CarbonRag V1.1.7 Chat UX Audit + Product Feel Polish

## Summary
- 本轮从当前 `main` 派生实验分支，建议分支名：`feature/v1.1.7-ux-audit-chat-polish`。本轮**不合并到 `main`**、不快进 `release/cloud-stable`。
- 现有 `ask/stream`、thinking 状态机、右侧抽屉、memory capsule 都已存在；V1.1.7 只补**还没做好**的部分，把聊天区从“功能工作台”推进到“更像主流 AI 聊天工具”的产品体验。
- 已锁定的实现决策：
  - Ask 页默认进入 **focus mode**
  - 右侧依据/系统状态继续用 `Drawer`，但默认更隐身
  - 新增 UI copy 必须保持中文，不引入新的英文界面文案
  - 本轮开发协同默认最多 **1 个子 agent**，仅用于只读研究或测试侧探查，不做多写者并发

## Key Changes
- 聊天工作台重排：
  - Ask 页默认 focus mode；focus mode 下全局 `AppShell` 进入轻壳层状态：
    - 左侧全局导航缩成更轻的 icon rail 或可收起窄栏
    - 顶部大标题和“企业试用/管理员入口”式说明文案降级为单行轻提示，不再抢聊天首屏
    - 管理员身份仍保留，但只保留小角色标识，不让正常聊天界面呈现后台感
  - Ask 页内部继续保留 session rail，但默认宽度收窄到约 `200~220px`，支持一键折叠到 mini rail
  - 中间消息列成为绝对主区，消息气泡最大宽度上调；右侧依据/状态不再常驻占宽

- Composer 重做：
  - 输入区改成真正的 sticky/floating 底部 composer，始终停留在可见区域
  - 主 row 只保留：
    - 附件按钮
    - 文本输入框
    - 发送按钮
  - scope、上传状态、知识条目状态压成紧凑 chips；知识条目管理入口、上下文细节、依据入口移到次级 row 或 popover
  - 输入区上方不再出现大块运行时解释文案；只保留轻量当前范围和进行中状态
  - 消息流底部补足安全留白，确保最后一条消息不被 composer 遮挡

- 消息流与 streaming 体验强化：
  - `user` 消息固定靠右、`assistant` 固定靠左、`system` 弱化为细条/胶囊样式
  - assistant 生成态固定为：
    - `pending`
    - `thinking`
    - `streaming`
    - `done`
    - `error`
  - 发送后用户消息立刻入流；assistant 占位气泡立即出现并经历完整状态迁移
  - thinking 可视状态增强：
    - 默认显示更明显的 pulse / dots / heartbeat
    - thinking 与 streaming 在文案、标签、动画上明确区分
    - 若收到 reasoning chunk，则在 assistant 气泡内以可折叠 thinking 区展示；未收到时也保留“思考中”占位
  - answer 继续逐 chunk 渲染，保证气泡在生成过程中明显“长出来”，而不是整段后刷

- 可读性和信息层级瘦身：
  - assistant 消息增加留白、段落节奏、列表间距，正文优先
  - trace id、上下文诊断、工程元数据下沉到 collapsible details 区，不再默认大面积显示
  - 反馈按钮弱化成次级动作，不与正文和依据入口争抢
  - citations 默认折叠为轻量来源摘要；完整依据只在点击后进入右侧 drawer 查看
  - 右侧 drawer 改成“选中消息驱动”的证据面板：
    - 默认只显示总数和来源分布
    - 选中一条 assistant 消息后聚焦该消息依据
    - 证据卡片只保留高扫描效率信息：标题、来源标签、短 snippet、展开/查看来源入口
  - 现有顶部 `memory_state` 大块说明压成单个上下文胶囊：
    - 默认只显示一个小占用指示 + 压缩状态短标签 + 一句 context source 摘要
    - 点击后才展开占用估算、最近轮次、摘要更新时间、压缩说明
  - context source 文案固定显性化，例如：
    - “本轮使用最近 6 轮 + 会话摘要 + 3 条依据”
    - “已压缩旧消息，保留最近 6 轮完整消息”

- 后端与契约只做补齐，不扩接口面：
  - 继续使用现有：
    - `POST /api/v1/sessions/{id}/ask`
    - `POST /api/v1/sessions/{id}/ask/stream`
  - 不新增新聊天 API
  - 本轮只冻结 `ask/stream` 的使用口径：
    - 前端默认走 `/ask/stream`
    - `/ask` 只作降级后备
  - 后端仅补稳定性缺口，不重写 orchestrator：
    - 保证 `status=thinking` 一定早于正文 delta
    - 保证 `metadata` 一定带 `citations`、`source_summary`、`memory_state`
    - 保证无 reasoning chunk 时前端仍能可靠进入 thinking 状态
  - memory 不重做体系；只在文档和前端呈现上把当前 `session_summary + 最近 6 轮 + grounding + 当前问题` 规则写死、讲清、显性化

## Public APIs / Interfaces
- 不新增新的业务 API。
- 继续沿用：
  - `POST /api/v1/sessions/{id}/ask`
  - `POST /api/v1/sessions/{id}/ask/stream`
- 前端默认行为冻结为：
  - Ask 页优先走 `/ask/stream`
  - `/ask` 仅作失败降级路径
- 本轮新增的“对外契约”主要是 UX 侧：
  - Ask 页默认 focus mode
  - Evidence / system status 默认非常驻，按需打开
  - Context diagnostics 默认折叠为胶囊

## Test Plan
- 前端 smoke：
  - Ask 页首屏打开后，聊天区明显成为视觉中心
  - 左侧 session rail 明显更窄，并支持折叠
  - 全局导航在 Ask 页 focus mode 下不再形成“第三条侧栏”压迫感
  - 输入区固定悬浮，滚动消息后仍始终可见
  - 用户消息靠右、AI 消息靠左、system 消息弱化
  - 发送后立即出现用户消息和 assistant 占位；assistant 依次进入 `pending -> thinking -> streaming -> done/error`
  - thinking 的动画和 streaming 的动画能明显区分
  - citations 默认折叠，不抢正文注意力；右侧 drawer 只在需要时展开
  - context 胶囊默认轻量，点开后才显示详细上下文信息
  - Ask 页 focus mode 下，管理员身份不会让聊天界面看起来像后台页

- 后端 / 契约回归：
  - `/api/v1/sessions/{id}/ask/stream` 继续通过现有 streaming 回归
  - provider 在“有 reasoning / 无 reasoning / provider error”三种情况下都能给出稳定事件序列
  - `metadata` 始终包含 `citations`、`source_summary`、`memory_state`
  - 非流式 `/ask` 保持兼容可用

- UX QA 交付：
  - 桌面正常宽度和更窄宽度各完成一轮 before/after 截图
  - QA checklist 明确记录：
    - 首屏理解成本
    - composer 可达性
    - streaming 感知
    - evidence 干扰度
    - context 透明度是否适度
    - admin 痕迹是否污染普通聊天体验

## Assumptions
- V1.1.7 从当前 `main` 派生实验分支开发，但本轮完成后**不合并回 `main`**、不更新 `release/cloud-stable`。
- 本轮只允许 **1 个子 agent** 协同；若实现期确需启用，也只能用于只读研究或测试侧探查，不允许多写者并发。
- 新增或修改的用户可见文案一律保持中文；英文只允许出现在研究材料引用或外部参考文档标题中。
- focus mode 默认开启，但本轮提供显式切换回标准工作台的入口；切换状态本轮不要求持久化。
- 本轮不扩后台、不扩知识任务流、不做新 business module、不重做 memory 体系、不做插件系统。

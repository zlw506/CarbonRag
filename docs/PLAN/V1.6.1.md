# V1.6.1 LangChain RAG Core Port

## Summary
- 本轮执行 `change-id: langchain-rag-core-port`，目标是把 CarbonRag RAG 主线从当前 BM25/disabled vector fallback 升级为 LangChain + Chroma + BM25 + HyDE + rerank 的真实 RAG 核心。
- 参考项目已在本地：`3rdparty/LangChain-RAG-FastAPI-Service-master/...`；其核心链路是 `RagService -> VectorStoreService -> Chroma/BM25 -> HyDE -> CrossEncoder rerank -> answer chain/tool`。
- 当前 CarbonRag `main` 仍默认 `RAG_ENGINE_ENABLED=false`、`RAG_VECTOR_ENABLED=false`，但已有 `knowledge_chunks`、`session_file_search`、`AskCitation.private_upload` 和 RAG Lab 基础，适合做“复刻级业务逻辑移植”，不推倒现有 auth/session/frontend。

## Key Changes
- 新增 `backend/app/langchain_rag/`，按参考项目结构移植为 CarbonRag 原生模块：`documents / embeddings / vector_store / bm25_store / splitter / hyde / reranker / retriever / answer_chain / service / tool / citations`。
- 接入依赖：`langchain`、`langchain-core`、`langchain-community`、`langchain-text-splitters`、`langchain-chroma`、`chromadb`、`rank-bm25`、`sentence-transformers`、`torch`、`langsmith`；优先按参考项目版本安装，若 pip resolver 失败则锁定最近兼容版本并记录。
- 默认打开比赛版 RAG：`.env.example` 改为 `RAG_ENGINE_ENABLED=true`、`RAG_VECTOR_ENABLED=true`、`RAG_LANGCHAIN_ENABLED=true`、`RAG_VECTOR_BACKEND=chroma`、`RAG_DEFAULT_MODE=mix`、`RAG_BM25_ENABLED=true`、`RAG_HYDE_ENABLED=true`、`RAG_RERANK_ENABLED=true`。
- 文档接入链路固定为：`file_parse_results / knowledge_chunks -> LangChain Document -> Chroma add_documents -> BM25 corpus refresh`，metadata 必须包含 `chunk_id / knowledge_item_id / file_id / owner_user_id / library_scope / source_type / title / page_number / sheet_name / slide_number / section_title / source_url`。
- 检索策略固定为：先 HyDE 生成 hypothetical query，失败则回退原 query 并写 warning；短查询 BM25 0.7 + vector 0.3，长查询 BM25 0.3 + vector 0.7，中等查询 0.5 + 0.5；最后 CrossEncoder rerank，模型不可用时显式 fallback，不允许静默假装已 rerank。
- Ask 接入改为默认走 `langchain_rag_search / langchain_rag_answer`：`public` 查公共政策，`private_sample` 查选中文件/私有知识，`mixed` 同时合并 public + private；旧 `app.rag` 作为兼容 fallback，但响应必须标明 fallback reason。
- 参考项目源码不整体提交；若直接复制/改写函数级代码，保留来源注释和授权说明。新增 `docs/research/rag/langchain-rag-fastapi-service-review.md` 记录复刻映射、授权状态、未移植内容。

## Public APIs / Interfaces
- 新增：
  - `GET /api/v1/rag/health`
  - `GET /api/v1/rag/index/stats`
  - `POST /api/v1/rag/index/rebuild`
  - `POST /api/v1/rag/index/file/{file_id}`
  - `POST /api/v1/rag/search`
  - `POST /api/v1/rag/answer`
- `POST /api/v1/rag/search` 返回 `query / hyde_query / hits / trace`，每个 hit 包含 `chunk_id / title / snippet / bm25_score / vector_score / rerank_score / source_type / file_id / page_number / sheet_name / slide_number`。
- `POST /api/v1/rag/answer` 返回 `answer / citations / retrieval_trace`；`/sessions/{id}/ask` 与 `/ask/stream` 同步透出 retrieval trace 到 metadata/source summary，前端可展示“HyDE、BM25、Vector、Rerank、引用来源数”。
- 权限边界：所有 RAG API 要求登录；普通用户只能索引/检索自己可见的 personal/shared/public 内容；管理员可 rebuild public/shared/all，但不能绕过正文访问边界读取普通用户私有内容。

## Test Plan
- 后端新增测试：`test_langchain_rag_documents.py`、`test_langchain_rag_chroma_index.py`、`test_langchain_rag_bm25.py`、`test_langchain_rag_hybrid_retriever.py`、`test_langchain_rag_hyde.py`、`test_langchain_rag_reranker.py`、`test_langchain_rag_answer.py`、`test_langchain_rag_user_isolation.py`、`test_ask_uses_langchain_rag.py`。
- 必测真实文件场景：上传政策 PDF 后能问条文并引用页码；上传企业电费 Excel 后能问某月用电并引用 sheet；`public/private_sample/mixed` 都走 LangChain RAG trace；非本人用户检索不到私有文件。
- 故障验收：Chroma 不可用时返回 `vector unavailable`，HyDE/rerank 失败必须有 warning，不允许假装成功；关闭 `RAG_LANGCHAIN_ENABLED` 时才允许旧 RAG fallback。
- 验证命令：`openspec validate langchain-rag-core-port --strict`、`openspec validate --all`、`cd backend && python -m pytest`、前端如展示 trace 则跑 `npm run typecheck && npm run build`。

## Assumptions
- 执行版本按本轮用户输入记为 `V1.6.1`，历史任务书里的 `V1.6.0` 作为项目总监原始措辞保留在计划说明中。
- 参考项目作者授权有效；仓库内只提交移植后的 CarbonRag 代码、研究映射和必要 attribution，不提交 `3rdparty` 原仓源码或 ZIP。
- 不移植参考项目的 Django/JWT/MySQL/Redis/Vue 前端；CarbonRag 保留现有 HttpOnly auth、session、AskPage、文件解析和知识库治理。

# CarbonRag v0.1.9A 计划

## 轮次名称
`calc-carbon + feedback foundation（local-first）`

## 本轮目标
本轮只做两条真实链路：

1. 落地 `POST /api/v1/calc-carbon` 的首个真实碳核算链路  
2. 给 ask + calc 两侧补统一反馈按钮与后端数据库落库

## 为什么本轮先不做上云
- 当前 VPS / 挂载方式仍在生成中
- 本轮先确保本地开发、测试和本地演示链路稳定
- 云端部署兼容性保留到下一轮，不在本轮强行并行推进

## 为什么 calc 本轮不接 ai_runtime carbon mode
- ask 链路已经稳定跑在 `ai_runtime`
- calc 这轮本质是“受控公式服务”，不是 LLM 推理能力
- 先把真实输入、因子加载、计算、解释、持久化和前端结果链路跑通
- 后续再把 carbon mode 和更复杂工具链统一接入运行时

## 本轮范围

### 做
- `backend/app/carbon/` 落地真实计算服务
- `data/factors/carbon_factors_v0_1_9a.json`
- `POST /api/v1/calc-carbon`
- 碳核算结果写入本地 SQLite
- `POST /api/v1/feedback`
- Ask + Calc 两侧赞/踩 + 可选备注
- 前端碳核算页从占位页推进为真实表单页

### 不做
- Netlify / VPS / 云端挂载验证
- Scope 3 全量
- 多行业模板
- 附件自动抽取活动数据
- `generate-report`
- 长期 memory
- 插件系统

## 验收口径
- `POST /api/v1/calc-carbon` 可正常返回结果
- breakdown、公式说明、因子 citations 都可展示
- ask 与 calc 都可提交反馈
- 反馈可写入本地 SQLite
- ask 现有 `public / private_sample / mixed` 不退化
- 后端 `pytest` 全绿
- 前端 `typecheck` 和 `build` 通过

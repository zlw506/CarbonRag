# CarbonRag V1.1.10 全屏工作台 + 主题系统建设轮

## Summary
- 当前左侧会话区已经上提到全局壳层，真正没修干净的是两件事：
  - 收起后主内容区仍保留展开态的宽度约束、留白和 padding 组合，所以 Ask / 碳核算 / 报告页没有真正吃满可用宽度。
  - 左侧会话区虽然有滚动容器，但壳层高度分配还不够硬，导致会话列表在某些情况下不能稳定滚到下方。
- V1.1.10 不再继续打补丁式改色，也不再重做会话栏位置。这个版本固定两条主线：
  - 先把 workbench 壳层做成真正的“收起即全屏放宽”的布局
  - 在现有 light/dark 基础上，建设完整的多主题引擎
- 本轮完成后，从 `feature/v1.1.10-theme-system-fullbleed` 快进到 `main`。不再继续把这轮停留在实验分支。

## Key Changes
- **壳层与全屏布局修正**
  - 保留当前全局左侧结构：品牌区、主导航、会话区、用户入口；不再恢复页面内第二条会话栏。
  - `AppShell` 改成真正的“单侧壳层 + 全宽内容区”模型：
    - 展开态：左侧固定工作台宽度，右侧主内容 `flex: 1 1 auto; min-width: 0; width: 0`
    - 收起态：左侧固定为纯图标宽度，右侧内容区去掉展开态残留的包裹宽度和大 padding
  - workbench 页面统一使用同一套全宽内容规则：
    - `Ask / 碳核算 / 报告页` 的主区都只保留一列主内容
    - 收起左栏后，主区直接放大，不再保留“像还留着半条会话栏”的空白
  - Ask 页继续保持右侧依据抽屉；抽屉不重新变回常驻栏

- **左侧会话列表滚动收口**
  - 会话区继续放在“报告生成”下面，但滚动责任写死：
    - `Sider` 固定 `100dvh`
    - 品牌区、主导航、会话区头部、用户入口固定
    - 只有会话列表 body 独立 `overflow-y: auto`
  - 会话列表始终可滚到底部，鼠标滚轮、触控板、触摸滚动都只作用在会话列表，不把整个页面顶出视口。
  - 收起态继续只剩图标，不显示会话缩略块；展开态会话列表只显示标题，不显示更新时间、消息数、附件数。

- **完整主题引擎**
  - 新增 `frontend/src/theme/` 作为主题层，至少包含：
    - `tokens.ts`
    - `presets/*.ts`
    - `antd-map.ts`
    - `storage.ts`
    - `provider.tsx`
  - 主题不再直接散落在 CSS 和 JSX 里，统一改成三层：
    - 语义 token
    - 主题预设
    - Ant Design token 映射
  - 语义 token 最少覆盖：
    - `background`
    - `backgroundSecondary`
    - `surface`
    - `surfaceElevated`
    - `border`
    - `text`
    - `textSecondary`
    - `primary`
    - `primaryHover`
    - `primaryActive`
    - `chatUserBubble`
    - `chatAssistantBubble`
    - `evidencePublic`
    - `evidencePrivate`
    - `evidenceCarbon`
    - `thinkingGlow`
  - 安装并使用 `@radix-ui/colors`，只在主题预设层取色，不把第三方色值直接散落进页面组件。
  - Ant Design 继续用 `ConfigProvider`，通过 `defaultAlgorithm / darkAlgorithm` + token 映射生成主题；自定义壳层、聊天气泡、抽屉、侧栏、高亮、引用标签继续走 CSS 变量。
  - 在 `frontend/index.html` 增加首屏预载脚本，先读本地主题设置并写入根节点属性，减少主题闪烁。

- **10 套主题预设 + 模式切换**
  - 预设固定实现这 10 套：
    - `carbon-blue`
    - `deep-space`
    - `slate-neutral`
    - `jade-green`
    - `teal-professional`
    - `indigo-focus`
    - `purple-research`
    - `rose-humanized`
    - `amber-warm`
    - `graphite-pro`
  - 每套都提供 `light` / `dark` 变体。
  - 模式固定为：
    - `light`
    - `dark`
    - `system`
  - 默认开放 4 套快速主题：
    - 碳蓝
    - 深空
    - 玉绿
    - 靛蓝专注
  - 其余 6 套放到“更多主题”面板中。
  - 主题选择入口直接放在左侧头像菜单内：
    - 头像弹层里放模式切换 + 4 套快速主题卡
    - “更多主题”打开主题抽屉，展示全部 10 套预设
  - 所有用户可见主题名和说明保持中文；内部 id 保持英文。

- **主题覆盖范围**
  - 第一轮必须完整覆盖：
    - AppShell
    - 左侧导航
    - 会话列表
    - Ask composer
    - 聊天气泡
    - 依据标签
    - 卡片
    - 抽屉
    - Button / Input / Tag / Menu / Card / Drawer 等常用 AntD 组件
    - 知识页、碳核算页、报告页
  - 目标不是“改一个主色”，而是让默认 4 套观感明显不同，并且浅色/深色都不割裂。

- **发布步骤**
  - 从当前实验基线切出 `feature/v1.1.10-theme-system-fullbleed`
  - 完成后做本地验证
  - 验证通过后 `main` 直接快进到 V1.1.10 分支提交并推远端
  - 这轮不额外引入新的业务模块或后端接口

## Public APIs / Interfaces
- 不新增后端 API。
- 前端新增主题状态契约：
  - `ThemeMode = light | dark | system`
  - `ThemePresetId = carbon-blue | deep-space | ... | graphite-pro`
- 本地持久化新增两项：
  - `carbonrag-theme-mode`
  - `carbonrag-theme-preset`
- 根节点新增主题属性，用于首屏预载和 CSS 变量切换：
  - `data-theme`
  - `data-theme-preset`
- 现有 Ask / 碳核算 / 报告接口不改，只有壳层和主题行为改变。

## Test Plan
- **布局与滚动**
  - Ask 页展开态：左侧显示主导航、会话区、用户入口；主聊天区正常
  - Ask 页收起态：左侧只剩图标；聊天主区宽度明显变大，不再保留展开态残余空白
  - 碳核算页和报告页同样验证收起后主区真实放宽
  - 左侧会话列表可滚到最底部；底部会话不会被顶出屏幕外
  - 页面本身不出现“全局滚动抢会话列表滚动”的问题

- **主题**
  - 10 套主题都能切换
  - 每套都能在 `light / dark / system` 下正常工作
  - 刷新页面后主题保持
  - `system` 模式下随系统主题变化而同步
  - 首屏不明显闪到错误主题
  - 默认 4 套主题的整体观感明显不同，不是只换一个按钮色

- **页面覆盖**
  - Ask、知识库、碳核算、报告页都应用同一主题系统
  - 聊天气泡、按钮、卡片、抽屉、标签、侧栏、引用区都随主题变化
  - 管理员和普通用户各验证一轮，确保布局差异不再和角色绑定

- **构建与回归**
  - `npm.cmd run typecheck`
  - `npm.cmd run build`
  - 手工截图保留：
    - 展开 / 收起左栏前后
    - 4 套默认主题浅色 / 深色
    - Ask / 碳核算 / 报告页各一张

## Assumptions
- V1.1.10 继续沿用现有 React 19 + Vite + Ant Design 5，不换 UI 框架。
- 主题选择器不单独开设置页，直接放在左侧头像菜单里；高级 6 套主题用抽屉承载。
- 收起态不再显示任何会话缩略块；只保留图标，这是为了解决“收起后仍不够宽”的根问题。
- 当前全局左侧会话区已经上提到壳层，这部分不重做位置，只修滚动、宽度和主题能力。
- `main` 合并方式默认采用快进，前提是 V1.1.10 在当前实验线基础上完成并通过本地验证。

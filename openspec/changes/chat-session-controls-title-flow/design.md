# Design

## Title Timing

`SessionService` 在用户消息落库后调用标题生成：

1. 第一次用户发送：根据当前用户问题生成临时标题。
2. 第二次用户发送：优先使用第一轮有效问答，加上第二次用户问题修正标题。
3. 第三次及以后：自动标题逻辑不再修改标题。

手动重命名始终开放；前两次自动命名仍允许覆盖手动标题，这是本轮明确的产品规则。

## Session Controls

会话控制保持用户隔离：普通用户和管理员都只能通过普通 session API 操作自己的会话。删除会话走数据库级级联删除消息、附件关联等子资源。

## Frontend

`SessionRail` 仅承担列表展示与菜单触发；真实操作由 `AppShell` 调用 session service 后刷新列表，避免和 AskPage 的会话上下文耦合。
